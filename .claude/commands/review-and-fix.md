---
description: feature-dev の code-reviewer エージェントでコードをレビューし、発見した問題を修正する。「レビューして修正して」でトリガー。
---

## User Input

```text
$ARGUMENTS
```

## Outline

1. **レビュー対象を特定する**
   - `$ARGUMENTS` にファイルパスや範囲が指定されていればそれを対象とする
   - 指定がない場合は、直近の変更ファイル（`git diff --name-only HEAD` または現在作業中のファイル）を対象とする

2. **`feature-dev:code-reviewer` エージェントでレビューを実行する**

   以下の指示でエージェントを起動する:

   ```
   以下のファイル（または変更差分）をレビューしてください。

   対象: [特定したファイルパスまたは範囲]

   確認項目:
   - バグ・ロジックエラー
   - セキュリティの脆弱性
   - コード品質の問題（重複・複雑度・可読性）
   - プロジェクトの慣習・命名規則への適合
   - テストの欠落または不適切なテスト

   信頼度が高い（修正すべき）問題のみを報告してください。
   各問題について以下を明記してください:
   - ファイルパスと行番号
   - 問題の種別と深刻度（CRITICAL / HIGH / MEDIUM / LOW）
   - 具体的な問題の説明
   - 推奨する修正内容
   ```

3. **レビュー結果を表示する**
   - エージェントの報告をそのまま表示する
   - 問題が0件の場合は「問題なし」と報告してスキップする

4. **修正を実行する**
   - CRITICAL / HIGH の問題から順番に修正する
   - 各修正後にテスト（`pytest tests/ -v` など）を実行して回帰がないことを確認する
   - MEDIUM / LOW の問題も可能な範囲で修正する
   - 修正できない問題（設計判断が必要など）はユーザーに報告して確認を求める

5. **修正サマリーを報告する**
   - 修正した問題の一覧（ID・ファイル・内容）
   - 未修正の問題とその理由
   - 最終テスト結果
