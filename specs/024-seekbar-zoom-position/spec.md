# Feature Specification: シークバーズーム時の現在位置インジケーター表示

**Feature Branch**: `024-seekbar-zoom-position`
**Created**: 2026-03-23
**Status**: Draft
**Input**: User description: "シークバーズーム時、現時点を表すポイントもズームしたシークバー上での位置を表現してほしい"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - ズーム中の現在位置を正確に把握する (Priority: P1)

ユーザーが AB ループ区間などを精密に設定するためにシークバーをズームした状態で再生中、現在の再生位置がズームされたシークバー上のどこにあるかを一目で確認できる。

ズーム前は全体の時間軸で位置を把握していたが、ズーム後は拡大された部分範囲内での正確な相対位置が分かるようになる。

**Why this priority**: ズーム機能の主目的は精密な区間操作であり、そのためには現在位置がズーム拡大表示上でも正しく示される必要がある。これがなければズーム機能の実用価値が大きく損なわれる。

**Independent Test**: 動画を再生しながらズームを有効にするだけでテスト可能。現在位置マーカーがズーム範囲内の正しい割合位置に表示されることを確認できる。

**Acceptance Scenarios**:

1. **Given** 動画が全体の 40% の位置で再生中でシークバーがズームされていない、**When** ズームを有効にして 30%〜50% の範囲を表示する、**Then** 現在位置マーカーがズームされたシークバーの中央付近（その範囲内の 50% 位置）に表示される
2. **Given** ズームが有効な状態で再生中、**When** 再生が進んで現在位置がズーム範囲の終端を超える、**Then** 現在位置マーカーはシークバーの右端に固定表示される
3. **Given** ズームが有効な状態、**When** ズームを解除する、**Then** 現在位置マーカーが動画全体の時間軸上の正しい位置に戻る

---

### User Story 2 - ズーム範囲外での現在位置の認識 (Priority: P2)

再生位置がズーム表示範囲の外にある場合、ユーザーがそれを視覚的に認識できる。

**Why this priority**: ズーム範囲外で再生中に位置が不明になることを防ぎ、ユーザーが状況を把握できるようにする。

**Independent Test**: 現在位置がズーム範囲外になるよう再生位置を設定してズームを有効にすることで独立してテスト可能。

**Acceptance Scenarios**:

1. **Given** 現在位置がズーム範囲より前にある状態、**When** ズームを有効にする、**Then** 現在位置マーカーはシークバーの左端に固定表示される
2. **Given** 現在位置がズーム範囲より後にある状態、**When** ズームを有効にする、**Then** 現在位置マーカーはシークバーの右端に固定表示される

---

### Edge Cases

- ズーム範囲が非常に狭い（例：100ms 以内）場合、現在位置マーカーは正常に表示されるか？
- 現在位置がちょうどズーム開始点または終了点と一致する場合、マーカーは端に正確に表示されるか？
- 動画未読み込み（duration = 0）状態でズームが有効な場合の挙動は？
- シークバーの表示幅が極端に小さい（数 px）場合のマーカー表示は崩れないか？

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: ズームモード有効時、現在の再生位置マーカーはズーム範囲（zoom_start〜zoom_end）内での相対割合に基づいてシークバー上に表示されなければならない
- **FR-002**: ズームモード無効時、現在の再生位置マーカーは動画全体の時間軸に基づく位置に表示されなければならない（既存動作を維持）
- **FR-003**: 現在位置がズーム範囲内にある場合、マーカーはシークバーのトラック左端〜右端の間に表示されなければならない
- **FR-004**: 現在位置がズーム範囲より前にある場合はマーカーを左端に、後にある場合は右端に固定表示しなければならない。このとき外観（色・スタイル）は通常時と同じとする
- **FR-005**: ズームモードの切り替え（有効化・無効化）時に、現在位置マーカーの位置が即座に更新されなければならない
- **FR-006**: 再生中に位置が変化するにつれて、ズームされたシークバー上の現在位置マーカーもリアルタイムで追従しなければならない

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ズームモード有効時、現在位置マーカーの表示位置がズーム範囲内の理論的な正確位置から視覚的に 1px 以内の誤差で表示される
- **SC-002**: ズーム有効化・無効化の切り替え直後の次の描画フレーム内に現在位置マーカーの位置が更新される（遅延なし）
- **SC-003**: ズーム範囲内で再生中、ユーザーが現在位置をシークバー上で常に視覚的に識別できる
- **SC-004**: 既存のズームなし状態での現在位置マーカー表示に変化がない（リグレッションなし）

## Clarifications

### Session 2026-03-23

- Q: ズーム範囲外のマーカーを端固定表示する際、通常時と外観を変えるか？ → A: 外観変更なし。端固定のみで十分（Option A）

## Assumptions

- 現在位置マーカーとは QSlider 標準のハンドル（つまみ）を指す。ズームモード時にはこのハンドルの表示位置をズーム範囲に合わせて調整する。
- ズーム範囲外の現在位置は端に固定表示する（非表示にはしない）。外観（色・スタイル）は通常時と同じとする。端への固定位置自体が「範囲外である」ことの視覚的シグナルとなる。
- 現在位置の更新頻度は既存の再生位置更新メカニズム（タイマーやプレーヤーイベント）に依存し、この機能では変更しない。
