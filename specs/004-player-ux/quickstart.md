# Quickstart: プレイヤー UX 改善 手動検証ガイド

**Branch**: `004-player-ux` | **Date**: 2026-03-15

---

## 前提条件

```bash
python main.py
```

動画ファイル（MP4 等）を 2〜3 本用意しておく。

---

## US1 — ドラッグ＆ドロップ

### 検証手順

1. アプリを起動する
2. ファイルマネージャーから動画ファイルをウィンドウにドラッグ＆ドロップする
3. **期待結果**: ファイルが即座に開かれ再生が始まる
4. 再生中に別の動画ファイルをドロップする
5. **期待結果**: 新しいファイルに切り替わって再生が始まる
6. `.txt` ファイルをドロップする
7. **期待結果**: 無視されて現在の再生が継続する

---

## US2 — 最近開いたファイル

### 検証手順

1. アプリを起動し、「ファイル > 開く」で動画を開く
2. 「ファイル > 最近開いたファイル」サブメニューを確認する
3. **期待結果**: ファイル名のみ表示、ホバーでフルパスがツールチップ表示される
4. アプリを終了して再起動する
5. 「ファイル > 最近開いたファイル」を確認する
6. **期待結果**: 前回のファイルが一覧に残っている
7. 一覧からファイルを選択する
8. **期待結果**: そのファイルが開かれて再生される

### エラーケース検証

- 存在しないファイルを一覧から選択 → エラーダイアログが出て一覧から消える

---

## US3 — ウィンドウリサイズ

### 検証手順

1. アプリを起動する
2. 1920×1080 の動画を開く
3. **期待結果**: ウィンドウが 16:9 のアスペクト比に調整される（約 100ms 以内）
4. 「表示 > ウィンドウを動画サイズに合わせる」を選択する
5. **期待結果**: ウィンドウが再調整される
6. 異なるアスペクト比の動画（例: 4:3）を開く
7. **期待結果**: ウィンドウがその動画の比率に再調整される

### エッジケース

- 低解像度動画（例: 320×240）→ 最小サイズ 800×600 を維持
- ユーザーが手動リサイズ後は自動リサイズが再発動しない

---

## US4 — フルスクリーン中カーソル非表示

### 検証手順

1. アプリを起動して動画を開く
2. ダブルクリックまたはメニューでフルスクリーンに切り替える
3. 3 秒間マウスを動かさない
4. **期待結果**: マウスカーソルが非表示になる
5. マウスを動かす
6. **期待結果**: カーソルが即座に再表示される（100ms 以内）
7. フルスクリーンを解除（Esc）して通常ウィンドウに戻る
8. 3 秒間マウスを動かさない
9. **期待結果**: カーソルは非表示にならない

### エッジケース

- フルスクリーン中にマウスを画面上部に移動 → メニューバーが自動表示される

---

## US5 — ブックマーク削除 Undo

### 検証手順

1. 動画を開き、AB ループでブックマークを 2 件作成する
2. 1 件目のブックマークの削除ボタンを押す
3. **期待結果**: ブックマークが一覧から消える
4. 3 秒以内に `Ctrl+Z` を押す
5. **期待結果**: 削除したブックマークが元の位置・全属性で復元される
6. 再度削除し、5 秒待ってから `Ctrl+Z` を押す
7. **期待結果**: 何も起こらない（復元されない）

---

## US6 — ブックマーク エクスポート＆インポート

### 検証手順

1. 動画を開き、ブックマークを 3 件作成する
2. 「ファイル > ブックマークをエクスポート」を選択する
3. **期待結果**: ファイル保存ダイアログが表示される
4. `bookmarks_test.json` として保存する
5. ブックマークをすべて削除する
6. 「ファイル > ブックマークをインポート」で `bookmarks_test.json` を選択する
7. **期待結果**: 3 件のブックマークが名前・区間・繰り返し回数込みで復元される

### エラーケース検証

- 無効な JSON ファイルをインポート → エラーメッセージが表示され既存データは変更されない
- 動画を開いていない状態でエクスポートメニューを確認 → メニューが無効化（グレー）されている
- 既に存在する JSON ファイル名でエクスポート → 上書き確認ダイアログが表示される
