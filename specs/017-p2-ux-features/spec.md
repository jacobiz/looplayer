# Feature Specification: P2 UX 機能群

**Feature Branch**: `017-p2-ux-features`
**Created**: 2026-03-18
**Status**: Draft
**Input**: F-105 ABループ区間のズーム表示 / F-401 設定画面（Preferences）/ F-501 初回起動オンボーディング / F-503 フルスクリーン中コントロールオーバーレイ

## User Scenarios & Testing *(mandatory)*

### User Story 1 — フルスクリーン中コントロールオーバーレイ（F-503）(Priority: P1)

楽器練習者・語学学習者がフルスクリーンで動画を見ているとき、マウスを画面下部に動かすと再生コントロール（シークバー・音量・再生/一時停止・A/B点セット・ABループトグル）がオーバーレイ表示され、キーボードを使わずに操作できる。3 秒間操作しないと自動で消え、映像に集中できる状態に戻る。

**Why this priority**: フルスクリーン中に操作不能になるのはクリティカルな UX 問題。ショートカットを知らない新規ユーザーがフルスクリーンから抜け出せない事例がある。

**Independent Test**: フルスクリーン状態でマウスを画面下端に移動するとコントロールバーが表示され、再生・シーク・音量調整が機能する。

**Acceptance Scenarios**:

1. **Given** フルスクリーン表示中、**When** マウスカーソルを画面下部 10% の領域に移動、**Then** コントロールバーがアニメーション付きで表示される
2. **Given** コントロールバーが表示中、**When** 3 秒間マウス操作がない、**Then** コントロールバーが自動的に非表示になる
3. **Given** コントロールバーが表示中、**When** シークバーをドラッグ、**Then** 通常ウィンドウ時と同等の精度でシークできる
4. **Given** コントロールバーが表示中、**When** 音量スライダーを操作、**Then** 音量が変更される
5. **Given** フルスクリーン表示中、**When** マウスカーソルが画面中央付近にある、**Then** コントロールバーは表示されない（映像に集中できる）
6. **Given** コントロールバーが表示中、**When** フルスクリーン解除ボタンを押す、**Then** 通常ウィンドウに戻る

---

### User Story 2 — 設定画面（Preferences）（F-401）(Priority: P2)

ユーザーが「ファイル > 設定」メニューから設定ダイアログを開き、現在メニューや JSON 手動編集に散在している全設定項目を一覧できる。カテゴリ分けされたパネルで各設定を変更・保存できる。

**Why this priority**: 機能が増えるにつれ設定の発見性が低下している。設定画面は今後の機能追加のための基盤にもなる。

**Independent Test**: 設定ダイアログを開き、再生終了時動作・アップデート確認・連続再生モードなどの既存設定が表示・変更できる。

**Acceptance Scenarios**:

1. **Given** アプリ起動後、**When** 「ファイル > 設定」を選択、**Then** 設定ダイアログが開く
2. **Given** 設定ダイアログが開いている、**When** 「再生」カテゴリを選択、**Then** 再生終了時動作・連続再生モード・エクスポートエンコードモードの設定が表示される
3. **Given** 設定ダイアログが開いている、**When** 「アップデート」カテゴリを選択、**Then** 起動時更新確認のオン/オフ設定が表示される
4. **Given** 設定値を変更後、**When** 「OK」ボタンを押す、**Then** 変更が即座に反映され、次回起動後も保持される
5. **Given** 設定値を変更後、**When** 「キャンセル」ボタンを押す、**Then** 変更が破棄され元の値に戻る
6. **Given** 設定ダイアログが開いている、**When** 「表示」カテゴリを選択、**Then** 常に最前面などの表示設定が表示される

---

### User Story 3 — 初回起動オンボーディング（F-501）(Priority: P3)

アプリを初めて起動したユーザーに、AB ループの基本フロー（動画を開く → A/B 点を設定 → ループ再生 → ブックマーク保存）を 4 ステップのオーバーレイチュートリアルで案内する。「次回から表示しない」チェックを付けて閉じると以後は表示されない。

**Why this priority**: 新規ユーザーの学習コスト削減は重要だが、既存ユーザーへの影響がなく、他機能と独立して実装できる。

**Independent Test**: 初回起動時にオーバーレイが表示され、4 ステップを順に進めて完了できる。「次回から表示しない」チェック後の再起動でオーバーレイが表示されない。

**Acceptance Scenarios**:

1. **Given** アプリの初回起動（オンボーディング未完了）、**When** メインウィンドウが表示、**Then** チュートリアルオーバーレイが自動表示される
2. **Given** チュートリアルオーバーレイ表示中、**When** 「次へ」ボタンをクリック、**Then** 次のステップに進む（全 4 ステップ）
3. **Given** 最終ステップ表示中、**When** 「完了」ボタンをクリック、**Then** オーバーレイが閉じ、通常操作に移行する
4. **Given** チュートリアル完了済みのユーザー、**When** アプリを再起動、**Then** チュートリアルは表示されない
5. **Given** チュートリアル表示中、**When** 「スキップ」ボタンをクリック、**Then** 完了フラグが保存されオーバーレイが即座に閉じる（以後は自動表示されない）

---

### User Story 4 — ABループ区間のズーム表示（F-105）(Priority: P4)

楽器・発音練習者が長時間動画で 1〜3 秒の短い AB ループを精密に調整したいとき、「ズームモード」をトグルで切り替えると現在の AB 区間がシークバー全幅に拡大表示され、フレーム単位の微調整が容易になる。

**Why this priority**: パワーユーザー向けの精密操作機能。他のストーリーと完全独立しており、後から追加しやすい。

**Independent Test**: 短い AB 区間（5 秒以内）を設定した状態でズームモードに切り替えると、その区間がシークバー全幅に拡大表示され、ドラッグで A/B 点を調整できる。

**Acceptance Scenarios**:

1. **Given** AB 区間が設定済み、**When** ズームモードトグルをオン、**Then** AB 区間がシークバー全幅に拡大表示される（前後に 10% 程度の余白付き）
2. **Given** ズームモードオン中、**When** AB ドラッグ操作、**Then** フレーム単位の精密な調整が可能になる（通常モードより高い解像度）
3. **Given** ズームモードオン中、**When** AB 区間を変更、**Then** シークバーの拡大範囲が新しい区間に追従する
4. **Given** ズームモードオン中、**When** ズームモードトグルをオフ、**Then** 通常のシークバー表示に戻り現在位置が正しく表示される
5. **Given** AB 区間が未設定、**When** ズームモードトグルをオン、**Then** トグルは無効のまま操作を受け付けない
6. **Given** ズームモードオン中、**When** 動画を別ファイルに切り替え、**Then** ズームモードが解除される

---

### Edge Cases

- フルスクリーン中にコントロールバーが表示された状態でウィンドウモードに戻った場合、コントロールバーは通常 UI に置き換わるため非表示になる
- 設定ダイアログを開いたまま動画が終端に達した場合、再生終了時動作の変更はダイアログを閉じた後の次回に適用される
- オンボーディング表示中にファイルをドラッグ&ドロップした場合、オーバーレイを自動的に閉じてドロップを処理する（この場合も完了フラグは保存しないため次回起動時に再表示）
- ズームモードで AB 区間の長さが全動画長の 1% 未満の極端なケース（最低表示幅を保証）
- 設定ダイアログの全入力は選択式（ドロップダウン・チェックボックス）のため無効値は発生しない

## Requirements *(mandatory)*

### Functional Requirements

#### F-503: フルスクリーン中コントロールオーバーレイ

- **FR-101**: フルスクリーン中にマウスカーソルが画面下端から 10% の領域に入ったとき、コントロールバーが表示されなければならない
- **FR-102**: コントロールバーは最後のマウス操作から 3 秒後に自動的に非表示にならなければならない
- **FR-103**: コントロールバーにはシークバー・再生/一時停止ボタン・音量スライダー・A点セットボタン・B点セットボタン・ABループトグルボタン・フルスクリーン解除ボタンが含まれなければならない
- **FR-104**: コントロールバー上でのマウス操作中はタイムアウトタイマーがリセットされなければならない
- **FR-105**: コントロールバー表示中はマウスカーソルを表示し続けなければならない（カーソル自動非表示との連動）

#### F-401: 設定画面

- **FR-201**: 「ファイル」メニューに「設定」メニュー項目が追加されなければならない
- **FR-202**: 設定ダイアログは「再生」「表示」「アップデート」の最低 3 カテゴリを持たなければならない
- **FR-203**: 既存の全設定項目（再生終了時動作・連続再生モード・エクスポートエンコードモード・起動時更新確認）が設定ダイアログ内に集約されなければならない
- **FR-204**: 設定変更は「OK」ボタン押下時に保存されダイアログが閉じられなければならない
- **FR-205**: 「キャンセル」ボタン押下で変更が破棄されダイアログが閉じられなければならない（「適用」ボタンは設けない）
- **FR-206**: 設定はアプリ再起動後も保持されなければならない

#### F-501: 初回起動オンボーディング

- **FR-301**: アプリ初回起動時（設定に完了フラグがない場合）にチュートリアルオーバーレイを自動表示しなければならない
- **FR-302**: チュートリアルは「動画を開く」「A/B 点を設定」「ループ再生」「ブックマーク保存」の 4 ステップで構成されなければならない
- **FR-303**: 各ステップには操作手順の説明テキストとステップ番号表示が含まれなければならない
- **FR-304**: 「スキップ」ボタンを提供しなければならない（スキップ時に完了フラグを保存する）
- **FR-305**: チュートリアル完了またはスキップ後にのみ完了フラグを設定に保存しなければならない（途中でアプリを終了した場合はフラグを保存せず、次回起動時にステップ 1 から再表示する）
- **FR-306**: 「ヘルプ」メニューからチュートリアルを手動で再表示できなければならない

#### F-105: ABループ区間のズーム表示

- **FR-401**: AB 区間が設定済みの場合のみ、ズームモードのトグルボタンが有効になければならない
- **FR-402**: ズームモードでは AB 区間がシークバー全幅に拡大表示され、区間前後に余白（各 10% 程度）を含まなければならない
- **FR-403**: ズームモード中の AB ドラッグ操作は通常モードより高精度（実質的にフレーム単位）で可能でなければならない
- **FR-404**: ズームモード中に AB 区間が変更された場合、表示範囲が自動更新されなければならない
- **FR-405**: 動画を別ファイルに切り替えた場合、ズームモードは自動的に解除されなければならない

### Key Entities

- **設定項目（Setting）**: キー・値・カテゴリ・表示名を持つ。既存の `settings.json` に永続化
- **オンボーディング状態**: `onboarding_shown` フラグ（bool）。設定に保存。ヘルプメニューからリセット可能
- **ズームモード状態**: 現在の拡大表示範囲（開始・終了時刻）。セッション中のみ保持（永続化不要）

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: フルスクリーン中にマウスを画面下端に動かしてから 300ms 以内にコントロールバーが表示される
- **SC-002**: フルスクリーン中の全コントロール操作（再生・シーク・音量）がキーボードなしで完了できる
- **SC-003**: 設定ダイアログで全既存設定項目が 1 画面から変更でき、メニューを探し回る必要がなくなる
- **SC-004**: 初回起動ユーザーがチュートリアルに従って最初のブックマークを 5 分以内に作成できる
- **SC-005**: 「次回から表示しない」でスキップしたユーザーへのチュートリアル再表示が 0 件
- **SC-006**: ズームモードで 5 秒以内の AB 区間において、フレーム単位の精密な A/B 点調整が直感的に行える（ドラッグ操作 1 回で目的のフレームに到達できる）

## Assumptions

- F-503 コントロールバーのレイアウトは通常ウィンドウのコントロールパネルを流用する（完全新規設計は不要）
- F-401 設定ダイアログは既存の `AppSettings` クラスを介して読み書きする（新しい永続化機構は不要）
- F-501 オンボーディングは非モーダルオーバーレイとし、背後のアプリ操作をブロックしない
- F-105 ズームモードはセッション内の一時状態であり、再起動時にリセットされる
- 日英 UI 切替（i18n）はすべての新規 UI 文字列に適用する

## Dependencies

- F-503 はカーソル自動非表示機能（spec 004 US4）との連動が必要
- F-401 設定画面は他機能の設定項目を収容する基盤となるため、他機能より先に実装することが望ましい
- F-501 は F-401 の `onboarding_shown` 設定フラグに依存する
- F-105 は既存の `BookmarkSlider` ウィジェット（シークバー）の拡張である

## Clarifications

### Session 2026-03-18

- Q: フルスクリーン中コントロールバーの表示トリガー領域 → A: 画面下端から 10%
- Q: 設定画面は既存メニュー項目を削除して一元化するか → A: 段階的移行とし既存メニュー項目を残しつつ設定ダイアログにも同項目を表示
- Q: オンボーディングをスキップした後に再表示できるか → A: 「ヘルプ > チュートリアルを表示」から手動再表示できる（FR-306）
- Q: フルスクリーンオーバーレイに AB ループ操作系ボタンを含めるか → A: 含める（シークバー・再生/停止・音量・A/B点セット・ABループトグル・フルスクリーン解除）
- Q: 設定ダイアログのボタン構成 → A: OK（保存して閉じる）＋ キャンセル（破棄して閉じる）の 2 ボタンのみ（適用ボタンは設けない）
- Q: チュートリアル途中終了後の次回起動時の動作 → A: ステップ 1 から再表示する（完了フラグは完了/スキップ時のみ保存）
- Q: 「次回から表示しない」チェックボックスの扱い → A: スキップボタンが完了フラグ保存+閉じる の役割を担うため、別途チェックボックスは不要（YAGNI）。FR-304 を更新済み
- Q: FR-303 の「対象 UI 要素ハイライト表示」の実装範囲 → A: 非モーダル QWidget オーバーレイで任意 UI 要素をハイライトする実装は過度な複雑さになるため、このイテレーションではスコープ外とする。説明テキスト+ステップ番号のみで代替
