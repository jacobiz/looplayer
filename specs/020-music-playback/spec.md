# Feature Specification: 音楽ファイル再生対応

**Feature Branch**: `020-music-playback`
**Created**: 2026-03-20
**Status**: Draft
**Input**: User description: "音楽ファイル再生にも対応したい"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 音楽ファイルを開いて再生する (Priority: P1)

ユーザーが MP3・FLAC・AAC などの音楽ファイルを開いたとき、アプリが正常に再生を開始する。ファイルを開くダイアログには音楽ファイルの拡張子も表示され、選択できる。

**Why this priority**: 音楽再生の基本機能。これがないと他の機能も成り立たない。

**Independent Test**: ファイルを開くダイアログから .mp3 ファイルを選択し、再生ボタンで音声が流れることを確認する。

**Acceptance Scenarios**:

1. **Given** アプリが起動している, **When** 「ファイルを開く」ダイアログで .mp3 ファイルを選択する, **Then** ファイルが読み込まれ再生が開始される
2. **Given** 音楽ファイルを再生中, **When** 再生・一時停止・シーク操作を行う, **Then** 動画ファイルと同様に正常に動作する
3. **Given** ファイルを開くダイアログを開く, **When** ファイルフィルタを確認する, **Then**「すべてのメディア」「動画ファイル」「音楽ファイル」の 3 種類のフィルタが存在する

---

### User Story 2 - 音楽再生中の UI 表示 (Priority: P2)

音楽ファイル再生中は映像がないため、映像エリアにプレースホルダー表示（音符アイコンや背景色）を表示し、ユーザーが「音楽を再生中」であることを視覚的に把握できる。

**Why this priority**: 現状の実装では映像エリアが真っ黒になるだけで UX が悪い。音楽再生中であることを明示することでユーザーの混乱を防ぐ。

**Independent Test**: .mp3 ファイルを開いた状態で映像エリアを確認し、プレースホルダーが表示されることを確認する。

**Acceptance Scenarios**:

1. **Given** 音楽ファイル（映像なし）を再生中, **When** 映像エリアを見る, **Then** 音楽再生中であることを示すプレースホルダー表示（音符アイコン等）が表示される
2. **Given** 音楽ファイルから動画ファイルに切り替えた, **When** 動画を開く, **Then** 映像エリアは通常の動画表示に戻る

---

### User Story 3 - 音楽ファイルのドラッグ&ドロップとプレイリスト (Priority: P3)

音楽ファイルをウィンドウにドラッグ&ドロップして開けること、またフォルダをドロップしたときに音楽ファイルもプレイリストに含まれること。

**Why this priority**: 動画と同様の操作性を提供することでシームレスな利用体験を実現する。

**Independent Test**: .mp3 ファイルをウィンドウにドラッグ&ドロップして再生が開始されることを確認する。

**Acceptance Scenarios**:

1. **Given** アプリが起動している, **When** .mp3 ファイルをウィンドウにドラッグ&ドロップする, **Then** ファイルが開かれ再生が始まる
2. **Given** 音楽ファイルと動画ファイルが混在するフォルダをドラッグ&ドロップする, **When** フォルダを開く, **Then** 両方のファイルタイプがファイル名順でプレイリストに追加される
3. **Given** 音楽ファイルのみのフォルダをドラッグ&ドロップする, **When** フォルダを開く, **Then** 音楽ファイルがプレイリストとして読み込まれ連続再生できる

---

### User Story 4 - 音楽ファイルへのブックマーク (Priority: P4)

音楽ファイル再生中にもブックマーク（ループ区間）を設定でき、指定区間の繰り返し再生が機能する。

**Why this priority**: 本アプリの中核機能であるブックマーク・ループ再生を音楽にも適用することで、語学学習や楽器練習など音楽用途での活用が広がる。

**Independent Test**: 音楽ファイルを開き、A-B ループブックマークを設定して、区間が繰り返し再生されることを確認する。

**Acceptance Scenarios**:

1. **Given** 音楽ファイルを再生中, **When** ブックマーク（A-B ループ）を設定する, **Then** 指定区間が繰り返し再生される
2. **Given** 音楽ファイルにブックマークが設定されている, **When** アプリを再起動して同じファイルを開く, **Then** ブックマークが保持されている

---

### Edge Cases

- 音声トラックのない動画ファイルを開いた場合は従来通り動画として扱う
- 破損した音楽ファイルを開こうとした場合は適切なエラーメッセージを表示する
- 非常に長い楽曲（数時間）でもシーク・タイムライン表示が正常に動作する
- 音楽ファイルと動画ファイルが混在するプレイリストで連続再生する際、映像エリアの表示が正しく切り替わる

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは MP3, FLAC, AAC, WAV, OGG, M4A, OPUS 形式の音楽ファイルを再生できなければならない
- **FR-002**: ファイルを開くダイアログは「すべてのメディア」統合フィルタ・「動画ファイル」フィルタ・「音楽ファイル」フィルタの 3 種類を提供しなければならない
- **FR-003**: ドラッグ&ドロップで音楽ファイルを受け付けなければならない
- **FR-004**: フォルダをドラッグ&ドロップした際、音楽ファイルをプレイリストに含め、ファイル名順（動画ファイルと混在する場合も同一の並び順）で並べなければならない
- **FR-005**: 音楽ファイル再生中は映像エリアに視覚的フィードバック（プレースホルダー）を表示しなければならない
- **FR-006**: 音楽ファイル再生中もブックマーク・ループ再生機能が動作しなければならない
- **FR-007**: 音楽ファイルの再生位置が記憶されなければならない（既存の再生位置記憶機能の拡張）
- **FR-008**: 音楽ファイルも「最近開いたファイル」に記録されなければならない
- **FR-009**: 音楽ファイルと動画ファイルが混在するプレイリストで連続再生ができなければならない
- **FR-010**: 音楽ファイル再生中はウィンドウタイトルにファイル名を表示しなければならない

### Key Entities

- **AudioFile**: 音楽ファイルを表す概念。対応拡張子（.mp3, .flac, .aac, .wav, .ogg, .m4a, .opus）を持つ。ファイルタイプは拡張子のみで判定する
- **MediaFile**: 動画ファイルと音楽ファイルを統合した概念。ファイルタイプ（video/audio）を拡張子ベースで属性として持ち、表示モードの切り替えに利用される

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 対応音楽形式（MP3, FLAC, AAC, WAV, OGG, M4A, OPUS）のすべてでファイルを開いてから再生開始まで 3 秒以内に完了する
- **SC-002**: 音楽ファイルのドラッグ&ドロップが 100% のケースで認識される（既存の動画ファイルと同等の動作）
- **SC-003**: 音楽ファイルに設定したブックマークが次回起動後も 100% 保持されている
- **SC-004**: 音楽・動画混在プレイリストで連続再生中に映像エリアが正しく切り替わる割合が 100% である
- **SC-005**: 既存の動画再生機能のテストがすべてパスする（リグレッションなし）

## Clarifications

### Session 2026-03-20

- Q: ファイルを開くダイアログのフィルタ設計はどうするか？ → A: 「すべてのメディア」統合フィルタ＋「動画ファイル」「音楽ファイル」個別フィルタの 3 種類を提供する
- Q: 音楽ファイルの判定方法はどうするか？ → A: 拡張子のみで判定（既存の仕組みと同じアプローチ）
- Q: 混在フォルダ読み込み時のプレイリスト並び順はどうするか？ → A: ファイル名順（既存の動画プレイリストと同じ並び順）

## Assumptions

- 再生エンジン（VLC）は対象音楽フォーマットをネイティブにデコードできる（追加ライブラリ不要）
- アルバムアートの表示はこのフィーチャーのスコープ外とし、プレースホルダー表示のみ実装する
- 音楽ファイルへのクリップ書き出し機能はこのフィーチャーのスコープに含めない
- 音楽ファイルのメタデータ（タイトル・アーティスト）の表示はこのフィーチャーのスコープ外とする
