# Feature Specification: Windows スタンドアロンアプリ インストーラ

**Feature Branch**: `006-windows-installer`
**Created**: 2026-03-16
**Status**: Draft
**Input**: User description: "Windows のスタンドアロンアプリとしてインストーラを作成したい"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - インストーラを実行してアプリをインストールする (Priority: P1)

Windows を使うエンドユーザーが、Python 等の事前インストールなしに LoopPlayer をダウンロード・インストールして起動できる。インストーラ（.exe）をダブルクリックするだけでセットアップが完了し、デスクトップのショートカットからアプリを起動できる。

**Why this priority**: Python 環境なしで使えることが「スタンドアロン」の核心価値であり、ユーザー獲得に直結する。

**Independent Test**: インストーラ .exe を Python 未インストールの Windows 環境で実行し、デスクトップショートカットから LoopPlayer が起動することを確認することで単独テスト可能。

**Acceptance Scenarios**:

1. **Given** Python が未インストールの Windows 10/11 PC、**When** インストーラ .exe をダブルクリックしてウィザードに従う、**Then** LoopPlayer がインストールされデスクトップにショートカットが作成される
2. **Given** インストール完了後、**When** デスクトップショートカットをダブルクリックする、**Then** LoopPlayer が正常に起動し動画を再生できる
3. **Given** インストール済みの状態、**When** スタートメニューで "LoopPlayer" を検索する、**Then** アプリのショートカットが表示されてクリックで起動できる

---

### User Story 2 - アプリをアンインストールする (Priority: P2)

ユーザーが Windows 標準の「アプリと機能」または「プログラムの追加と削除」から LoopPlayer をアンインストールできる。

**Why this priority**: Windows アプリとして適切な市民権を持つために必要。インストーラがあれば通常ペアで提供される機能。

**Independent Test**: インストール後に「アプリと機能」からアンインストールを実行し、アプリフォルダとショートカットが削除されることを確認することで単独テスト可能。

**Acceptance Scenarios**:

1. **Given** LoopPlayer がインストール済み、**When** 「アプリと機能」から LoopPlayer を選択してアンインストールを実行する、**Then** アプリファイル・ショートカット・スタートメニューエントリが削除される
2. **Given** アンインストール後、**When** ユーザーデータ（`~/.looplayer/`）の扱いを確認する、**Then** ブックマーク等のユーザーデータは保持される（アプリファイルのみ削除）

---

### User Story 3 - 同じ PC で新バージョンに更新する (Priority: P3)

既存インストールがある PC で新バージョンのインストーラを実行したとき、旧バージョンが上書きインストールまたは自動アンインストール後にインストールされる。

**Why this priority**: バージョンアップの運用が複雑になることを防ぐ。

**Independent Test**: 旧バージョンインストール後に新バージョンのインストーラを実行し、新バージョンが起動すること・ユーザーデータが保持されることを確認で単独テスト可能。

**Acceptance Scenarios**:

1. **Given** 旧バージョンがインストール済み、**When** 新バージョンのインストーラを実行する、**Then** 新バージョンに更新され、ブックマーク等のユーザーデータは保持される

---

### Edge Cases

- Python や VLC が別途インストールされている PC でインストーラを実行した場合も正常動作すること
- インストール先ドライブの空き容量が不足している場合、エラーメッセージを表示してインストールを中断すること
- 管理者権限なしでインストーラを実行した場合、ユーザースコープ（`%LOCALAPPDATA%`）へのインストールにフォールバックすること
- ウイルス対策ソフトによるスマートスクリーン警告が表示される場合があること（未署名 .exe の制約）
- インストールが途中で失敗した場合（権限エラー・予期しない障害など）、コピー済みのファイル・ショートカットをすべて自動削除してクリーンな状態にロールバックすること

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: インストーラは Python・VLC・その他依存ライブラリをすべて同梱し、追加インストールなしで動作しなければならない
- **FR-002**: インストーラは Windows 10 および Windows 11（64bit）で動作しなければならない
- **FR-003**: インストール後、デスクトップとスタートメニューにショートカットを作成しなければならない
- **FR-004**: アンインストールは Windows 標準の「アプリと機能」から実行できなければならない
- **FR-005**: アンインストール時にユーザーデータ（`~/.looplayer/`）は削除してはならない
- **FR-006**: インストール先ディレクトリをウィザード上でユーザーが変更できなければならない（デフォルト: `%LOCALAPPDATA%\LoopPlayer`）
- **FR-007**: インストール完了後、LoopPlayer を今すぐ起動するオプションをウィザード上で提供しなければならない
- **FR-008**: インストーラのウィザード UI は日本語・英語の 2 言語に対応し、Windows のシステム言語が日本語の場合は日本語、それ以外は英語で表示されなければならない
- **FR-009**: 既存バージョンがある場合、上書きインストールを行えなければならない
- **FR-010**: インストーラは単一の .exe ファイルとして配布できなければならない
- **FR-011**: インストールが途中で失敗した場合、コピー済みファイル・ショートカット・レジストリエントリをすべて自動削除してロールバックしなければならない
- **FR-012**: インストーラは GitHub Releases にリリース成果物として添付できる形式で生成しなければならない

### Key Entities

- **インストーラ (.exe)**: Python ランタイム・依存ライブラリ・アプリ本体をバンドルした自己実行形式の配布物
- **アプリケーションディレクトリ**: インストール先フォルダ（デフォルト `%LOCALAPPDATA%\LoopPlayer`）
- **ショートカット**: デスクトップ・スタートメニューに作成されるランチャー
- **ユーザーデータ**: `~/.looplayer/` 以下のブックマーク・履歴ファイル（インストーラ管理外、保持対象）

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: Python 未インストールの Windows 10/11 環境でインストーラを実行し、5 分以内にアプリが起動できる
- **SC-002**: インストーラの .exe ファイルサイズが 200MB 以下である
- **SC-003**: 技術知識のないユーザーがインストーラの追加手順なしに起動まで完結できる（ウィザードの指示のみで完了）
- **SC-004**: アンインストール後にユーザーデータ（ブックマーク・履歴）が残存している
- **SC-005**: クリーン環境でのインストール成功率が 95% 以上である

## Clarifications

### Session 2026-03-16

- Q: インストールが途中で失敗した場合、中途半端にコピーされたファイルを自動削除してロールバックするか → A: 完全ロールバック（失敗時はコピー済みファイル・ショートカットをすべて自動削除）
- Q: インストーラはどこで配布するか → A: GitHub Releases（バージョンタグをトリガーにリリースページへ添付）
- Q: インストーラ UI の言語は日本語固定か、システム言語対応か → A: 日本語・英語の 2 言語対応（Windows のシステム言語で自動切替）

## Assumptions

- ターゲットアーキテクチャは Windows 64bit のみとする
- インストールにはユーザーレベルの権限（管理者不要）を想定する
- コード署名は初期リリースでは未署名とし、スマートスクリーン警告はユーザーが許容する運用とする
- バージョンタグプッシュをトリガーとした GitHub Actions リリースワークフロー（`release.yml`）の作成は本フィーチャーのスコープに含む。PR ごとの継続的インテグレーションビルドはスコープ外
- アプリのバージョン番号は別途バージョンファイルで管理するものとする
