# Feature Specification: Bookmark Side Panel Toggle

**Feature Branch**: `021-bookmark-sidepanel`
**Created**: 2026-03-21
**Status**: Draft
**Input**: User description: "ブックマーク一覧を表示切り替えでサイドパネルに表示したい"

## User Scenarios & Testing *(mandatory)*

### User Story 1 — サイドパネルの表示・非表示切り替え (Priority: P1)

ユーザーはメニューまたはキーボードショートカットで、ブックマーク一覧サイドパネルの表示・非表示をワンアクションで切り替えられる。

**Why this priority**: 機能の核心。パネルを左右に並べるレイアウト変更と、表示切り替え操作が揃ってはじめて価値を持つ。

**Independent Test**: アプリを起動してメニューまたはショートカット操作を実行するだけで、パネルの出現・消滅を確認でき独立して検証できる。

**Acceptance Scenarios**:

1. **Given** アプリ起動直後（サイドパネル非表示）、**When** メニュー「表示 → ブックマークパネル」を選択、**Then** 動画フレーム右側にブックマーク一覧パネルが表示される
2. **Given** サイドパネルが表示中、**When** 同じメニュー項目またはショートカットを実行、**Then** パネルが非表示になり動画エリアが横幅いっぱいに拡大する
3. **Given** サイドパネルが非表示、**When** キーボードショートカット（`B` キー）を押す、**Then** サイドパネルが表示される
4. **Given** サイドパネルが表示中、**When** `B` キーを押す、**Then** サイドパネルが非表示になる
5. **Given** サイドパネルが表示中、**When** メニュー項目を確認、**Then** 「ブックマークパネル」項目にチェックマークが表示されている

---

### User Story 2 — サイドパネルの幅調整と永続化 (Priority: P2)

ユーザーはサイドパネルと動画エリアの境界をドラッグして幅を自由に調整でき、次回起動時に同じ幅が維持される。

**Why this priority**: 幅の好みは人それぞれ。小さいパネルで動画を広く使いたいユーザーと、ブックマーク詳細を広く見たいユーザーどちらにも対応できる。

**Independent Test**: パネルを表示してドラッグで幅を変更し、アプリを再起動して幅が保持されているかのみで検証できる。

**Acceptance Scenarios**:

1. **Given** サイドパネルが表示中、**When** パネルと動画の境界線をドラッグ、**Then** 任意の幅にリサイズできる
2. **Given** サイドパネルの幅を変更済み、**When** アプリを終了して再起動しパネルを表示、**Then** 前回変更した幅で表示される
3. **Given** 初回起動でデフォルト幅未設定、**When** サイドパネルを表示、**Then** デフォルト幅 280px（最小 240px）で表示される

---

### User Story 3 — フルスクリーン時のパネル自動制御 (Priority: P3)

フルスクリーンモード中はサイドパネルが自動的に非表示になり、フルスクリーン終了後に表示状態が元に戻る。

**Why this priority**: 映像視聴に集中したいフルスクリーン体験を損なわないための品質要件。

**Independent Test**: フルスクリーン切り替えの前後でパネル表示状態を確認するだけで独立して検証できる。

**Acceptance Scenarios**:

1. **Given** サイドパネルが表示中、**When** フルスクリーンに切り替える、**Then** サイドパネルが非表示になる
2. **Given** フルスクリーン中（元々パネル表示中だった）、**When** フルスクリーンを解除、**Then** サイドパネルが再び表示される
3. **Given** サイドパネルが非表示中、**When** フルスクリーンに切り替えて解除、**Then** サイドパネルは非表示のまま維持される

---

### Edge Cases

- ブックマークが 0 件の場合でもパネル自体は表示できる（空のリストを表示）
- ウィンドウ幅が極端に狭い場合、サイドパネルは最小幅（240px）を下回らない
- パネルが表示中に動画を閉じた場合、パネルは表示を維持し空リストを表示する。次のファイルを開いた際にブックマーク一覧が自動更新される

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: ブックマーク一覧およびプレイリストを含むパネルタブウィジェットは、動画フレームの右側にサイドパネルとして配置されなければならない
- **FR-002**: ユーザーはメニュー「表示 → ブックマークパネル（`B`）」でサイドパネルを表示・非表示に切り替えられなければならない
- **FR-003**: メニュー項目はサイドパネルが表示中かどうかをチェックマークで示さなければならない
- **FR-004**: サイドパネルの表示状態（表示/非表示）はアプリ終了後も永続化されなければならない
- **FR-005**: サイドパネルの幅はドラッグで調整でき、その値はアプリ終了後も永続化されなければならない
- **FR-006**: サイドパネルのデフォルト幅は 280px（最小 240px）でなければならない。設定済みの幅がある場合はその値を使用する
- **FR-007**: フルスクリーン移行時にサイドパネルを自動的に非表示にし、フルスクリーン解除時に元の表示状態を復元しなければならない
- **FR-008**: サイドパネル内のすべてのブックマーク機能（選択・連続再生・書き出し・削除・タグ操作など）は変更なく動作しなければならない
- **FR-009**: 再生コントロール（シークバー・再生ボタン・AB ループ設定など）はサイドパネルの表示状態にかかわらず常に表示されなければならない

### Key Entities

- **サイドパネル表示状態**: 表示/非表示の真偽値。アプリ設定として永続化される。
- **サイドパネル幅**: ピクセル値（整数）。アプリ設定として永続化される。デフォルトは 280px（最小 240px）。設定済みの場合はその値を使用する。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ブックマークパネルの表示切り替えが 1 アクション（メニュー選択またはキー押下）で完了する
- **SC-002**: パネル表示・非表示の切り替えが視覚的に即座に反映される（PyQt6 のレイアウト更新は同期処理のため、アクション後の次フレームで反映される）
- **SC-003**: アプリを再起動した後、パネルの表示状態と幅が前回終了時と同一である
- **SC-004**: パネルが表示されている状態でも、すべてのブックマーク操作（ループ再生・書き出し・削除）が正常に実行できる
- **SC-005**: フルスクリーン切り替え前後でパネルの表示状態が仕様通りに制御される（100%）
- **SC-006**: 既存の統合テストがすべて通過する（リグレッションゼロ）

## Assumptions

- プレイリストパネルはブックマークパネルと同じタブウィジェット内に含まれているため、タブウィジェットをまとめてサイドパネルへ移動する。プレイリストタブは引き続き同じタブウィジェット内に表示される（動作は現状維持）。
- ショートカットキーは `B` キーとする（既存のショートカット一覧と競合なし）。
- `AppSettings` クラスにパネル表示状態と幅の 2 フィールドを追加して永続化する。
- アプリ起動時のデフォルトはサイドパネル非表示とする（既存の動作を踏襲）。
