# Feature Specification: シークバークリックシーク

**Feature Branch**: `008-seekbar-click-seek`
**Created**: 2026-03-16
**Status**: Draft
**Input**: User description: "シークバーをクリックした際に再生ポイントをそこまで移動したい"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - シークバートラッククリックでシーク (Priority: P1)

動画を再生中または一時停止中にシークバーのトラック（つまみ以外の領域）をクリックすると、クリックした位置に対応する再生位置へ即座にジャンプする。マウスボタンを押したままドラッグした場合も、ドラッグ中リアルタイムに再生位置が追従する。現在はシークバーのつまみをドラッグするか、キーボードショートカットでしかシーク位置を変えられないため、直感的なワンクリックシークができない。

**Why this priority**: シークバーをクリックして移動先に飛ぶのは動画プレイヤーとして最も基本的かつ期待される操作であり、他のすべての改善の前提となる。

**Independent Test**: 動画を開いた状態でシークバーの任意の点をクリックし、再生位置がクリック位置に比例した時刻へ移動することを確認することで独立してテスト可能。

**Acceptance Scenarios**:

1. **Given** 動画が再生中, **When** シークバーのトラック上の任意の点（つまみ以外）を左クリックする, **Then** 再生位置がクリック位置に対応する時刻へ即座にジャンプし、再生が継続する
2. **Given** 動画が一時停止中, **When** シークバーのトラック上の任意の点を左クリックする, **Then** 再生位置がクリック位置に対応する時刻へジャンプし、一時停止状態が維持される
3. **Given** シークバーの左端付近をクリックした, **When** クリック位置が動画の先頭付近に対応する, **Then** 再生位置は 0 ms またはその近傍へ移動する
4. **Given** シークバーの右端付近をクリックした, **When** クリック位置が動画の末尾付近に対応する, **Then** 再生位置は動画長に近い時刻へ移動する

---

### User Story 2 - ブックマークバー上のクリックは既存挙動を維持 (Priority: P2)

シークバー上に重ね描きされているブックマーク区間バーをクリックした場合は、既存のブックマーク選択操作が行われ、シークは発生しない。US1 の実装がこの既存挙動を壊さないことを保証する。

**Why this priority**: ブックマーク区間のクリック選択はすでにリリース済みの機能であり、US1 の追加によってリグレッションを起こしてはならない。

**Independent Test**: ブックマークが設定されている動画で、ブックマーク区間バーをクリックしたときにシークが発生せずブックマーク選択通知が発火することを確認する。

**Acceptance Scenarios**:

1. **Given** シークバーにブックマーク区間バーが表示されている, **When** ブックマーク区間バーを左クリックする, **Then** ブックマーク選択通知が発火し、再生位置は変化しない
2. **Given** シークバーにブックマーク区間バーが表示されている, **When** ブックマーク区間バー以外のトラック部分をクリックする, **Then** クリック位置に対応する再生位置へジャンプする（US1 の動作）

---

### Edge Cases

- 動画が読み込まれていない（duration = 0）状態でシークバーをクリックしても、シークは行われない
- クリック位置がトラックの左端を下回る場合は先頭（0 ms）として処理する
- クリック位置がトラックの右端を超える場合は末尾（duration ms）として処理する
- 左クリック以外（右クリック・中クリック）はシークに使用しない

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: シークバーのトラック上を左クリックしたとき、クリック位置に対応する再生位置（ミリ秒）へ再生ヘッドを移動させなければならない
- **FR-001b**: マウスボタンを押したままドラッグした場合、ドラッグ中リアルタイムにクリック位置に追従して再生位置を更新し続けなければならない
- **FR-002**: クリック位置から再生時刻への変換は、グルーブ（トラック描画領域）の幅に対するクリック位置の比率に動画長を乗じることで求めなければならない
- **FR-003**: ブックマーク区間バー上のクリックは従来どおりブックマーク選択通知を発火し、シークを行ってはならない
- **FR-004**: 動画が再生中の場合、シーク後も再生を継続しなければならない
- **FR-005**: 動画が一時停止中の場合、シーク後も一時停止状態を維持しなければならない
- **FR-006**: 動画が読み込まれていない（duration ≤ 0）場合、クリックによるシークは無視しなければならない
- **FR-007**: 変換後の再生位置は [0, duration] の範囲にクリップしなければならない

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: シークバーのトラックをクリックしてから再生位置が更新されるまでの遅延が、ユーザーが体感できない範囲に収まる
- **SC-002**: 既存のブックマーク区間バークリック機能のテストがすべてパスし、リグレッションが発生しない
- **SC-003**: クリック位置と実際のシーク位置のズレが、動画長の 1% 以内に収まる
- **SC-004**: 動画未読み込み時にクリックしてもクラッシュや例外が発生しない

## Clarifications

### Session 2026-03-16

- Q: ドラッグ中のシーク挙動（クリックのみか、ドラッグ追従か） → A: クリック時にシーク開始し、ドラッグ中もリアルタイムに追従する（Option B）

## Assumptions

- シークバーは既存の `BookmarkSlider` を引き続き使用する
- シーク通知は新たなシグナルを介して行い、シークバーウィジェットが直接再生エンジンを操作しない
- グルーブ（トラック描画）領域の取得は既存の仕組みを再利用する
