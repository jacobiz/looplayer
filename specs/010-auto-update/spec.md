# Feature Specification: 自動更新機能

**Feature Branch**: `010-auto-update`
**Created**: 2026-03-17
**Status**: Draft
**Input**: User description: "自動更新機能をつける"

## User Scenarios & Testing *(mandatory)*

### User Story 1 — 起動時に新バージョンを通知する (Priority: P1) 🎯 MVP

アプリを起動すると、バックグラウンドで最新バージョンを確認し、新しいリリースが存在する場合はダイアログでユーザーに知らせる。ユーザーは「今すぐダウンロード」か「あとで」を選択できる。

**Why this priority**: アップデートの存在をユーザーに伝えることが自動更新機能の最低限の価値であり、他のすべての機能の前提となる。

**Independent Test**: アプリ起動時に「新バージョンが利用可能」ダイアログが表示され、「今すぐ」か「あとで」を選択できることを確認することで、単独テスト可能。

**Acceptance Scenarios**:

1. **Given** 現在のバージョンより新しいリリースが公開されている状態で、**When** ユーザーがアプリを起動すると、**Then** 「バージョン X.Y.Z が利用可能です。今すぐダウンロードしますか？」というダイアログが表示される。
2. **Given** 最新バージョンがすでに使用中の状態で、**When** ユーザーがアプリを起動すると、**Then** 更新通知は一切表示されない（サイレント）。
3. **Given** 更新確認中にネットワークエラーが発生した状態で、**When** ユーザーがアプリを起動すると、**Then** エラーは静かに無視され、通常通り起動できる。
4. **Given** 更新通知が表示された状態で、**When** ユーザーが「あとで」を選択すると、**Then** ダイアログが閉じ、次回起動まで再表示されない。
5. **Given** 更新通知が表示された状態で、**When** ユーザーが「今すぐダウンロード」を選択すると、**Then** インストーラーのダウンロードが開始され進捗が表示される。ダウンロード完了後にインストーラーが自動起動し、アプリが終了する。
6. **Given** インストーラーのダウンロード中に、**When** ユーザーがキャンセルを選択すると、**Then** ダウンロードが中止され、アプリは通常動作に戻る。

---

### User Story 2 — 手動で更新確認を実行する (Priority: P2)

ユーザーがヘルプメニューから「更新を確認…」を選択することで、いつでも即座にアップデート確認を実行できる。

**Why this priority**: 起動時チェックを無効にしているユーザーや、特定のタイミングで確認したいユーザーのために、手動トリガーは必須の補完機能である。

**Independent Test**: ヘルプメニュー → 「更新を確認」を選択したときに最新バージョン情報が表示されることを確認することで、単独テスト可能。

**Acceptance Scenarios**:

1. **Given** アプリが起動している状態で、**When** ユーザーがヘルプメニュー → 「更新を確認…」を選択すると、**Then** 確認が実行され、最新の場合は「最新バージョンを使用中です」と表示される。
2. **Given** アプリが起動している状態で、**When** ユーザーが手動確認を実行し新バージョンが見つかると、**Then** US1 と同様の更新通知ダイアログが表示される。
3. **Given** ネットワーク接続がない状態で、**When** ユーザーが手動確認を実行すると、**Then** 「更新確認に失敗しました。インターネット接続を確認してください。」と表示される。

---

### User Story 3 — 起動時チェックの有効/無効を設定する (Priority: P3)

ユーザーが起動時の自動更新確認を有効または無効にできる。設定はアプリを終了しても保持される。

**Why this priority**: 低速な接続環境や組織内配布環境では、起動時ネットワーク通信を避けたいユーザーが存在する。P1・P2 が完成してから対応すれば十分。

**Independent Test**: 設定で無効化した後にアプリを再起動しても更新通知が表示されないことを確認することで、単独テスト可能。

**Acceptance Scenarios**:

1. **Given** 起動時チェックが有効（デフォルト）な状態で、**When** ユーザーがヘルプメニューの「起動時に更新を確認する」のチェックを外して再起動すると、**Then** 起動時に更新確認が行われず、通知も表示されない。
2. **Given** 起動時チェックが無効な状態で、**When** ユーザーがヘルプメニューの同項目を再びチェックして再起動すると、**Then** 次回起動から通知チェックが再開される。

---

### Edge Cases

- ネットワーク接続がない、またはタイムアウトした場合 → 起動を妨げずサイレントにスキップ
- バージョン確認先が 503/404 を返した場合 → 起動を妨げずサイレントにスキップ
- 更新確認が非常に遅い場合（低速回線） → メインウィンドウの表示をブロックしない（バックグラウンド処理）
- 現在使用中と同一バージョンの「更新」を確認した場合 → 通知しない
- オフラインのユーザーが手動確認した場合 → エラーメッセージを表示してアプリは継続動作
- インストーラーのダウンロード中に通信エラー・中断が発生した場合 → エラーメッセージをモーダルダイアログ内に表示し、「再試行」ボタンで手動リトライ可能。「キャンセル」でダイアログを閉じてアプリに戻る。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: アプリ起動時に最新バージョン情報をバックグラウンドで取得する。取得処理はメインウィンドウの表示をブロックしてはならない。
- **FR-002**: 新バージョンが利用可能な場合、現在のバージョンと新バージョンの番号を明示した通知ダイアログを表示する。
- **FR-003**: 通知ダイアログには「今すぐダウンロード」と「あとで」の 2 つの選択肢を提供する。
- **FR-004**: 「今すぐダウンロード」を選択した場合、アプリがインストーラーファイルを OS の一時フォルダにダウンロードし、完了後に自動起動する。ダウンロード中はモーダルダイアログ（進捗バー＋キャンセルボタン）を表示し、アプリの他の操作はブロックする。インストーラー起動後はアプリを終了する。ダウンロードファイルのクリーンアップは OS に委ねる。
- **FR-005**: ヘルプメニューに「更新を確認…」項目を追加し、いつでも手動確認を実行できる。
- **FR-006**: 更新確認はアプリ起動時のみ実行する（毎回）。起動後のバックグラウンド定期確認は行わない。
- **FR-007**: ネットワークエラー・タイムアウト時は起動を妨げず、エラーをサイレントに処理する（手動確認時のみエラーメッセージを表示する）。
- **FR-008**: ヘルプメニューにチェック付きメニュー項目「起動時に更新を確認する」を追加し、チェックの ON/OFF で自動確認を有効/無効に切り替えられる。設定は `~/.looplayer/settings.json` に即時保存し、次回起動から反映する。デフォルトは ON。
- **FR-009**: 現在使用中のバージョンがすでに最新の場合、起動時には何も表示しない（サイレント）。

### Key Entities

- **バージョン情報**: 現在インストール済みのバージョン番号、取得した最新バージョン番号、ダウンロード先 URL
- **更新設定**: 起動時自動確認の有効/無効フラグ（`~/.looplayer/settings.json` に既存設定と統合して永続化）

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 更新確認処理はバックグラウンドで実行され、アプリ起動からメインウィンドウ表示までの時間を増加させない（ブロックゼロ）。
- **SC-002**: 新バージョンが存在する場合、起動から 5 秒以内に通知ダイアログが表示される（通常回線環境下）。
- **SC-003**: 手動確認（ヘルプメニュー）を実行したとき、10 秒以内に結果（最新 or 新バージョン通知）が表示される。
- **SC-004**: ネットワーク障害時でもアプリが正常に起動・動作し続ける（更新確認失敗によるクラッシュ・フリーズゼロ）。
- **SC-005**: ユーザーが「あとで」を選択した場合、同一セッション中（次回起動まで）は再通知しない。

## Assumptions

- バージョン情報の取得先は GitHub Releases（現在のリリースフローと一致）を使用する。
- 現在のバージョンはコード内の既存のバージョン定数として管理されており、比較はセマンティックバージョニング（例：1.2.0 > 1.1.3）で行う。
- 起動時チェックはデフォルトで有効とする。
- チェックのタイムアウト上限は 5 秒とする（ネットワーク待機上限）。
- リリースノートの表示は今回のスコープ外とする。

## Clarifications

### Session 2026-03-17

- Q: 更新設定（起動時自動確認 ON/OFF フラグ）の永続化先は？ → A: 既存の `~/.looplayer/settings.json` に追記（新規ファイル不要）
- Q: インストーラーダウンロード中の UI は？ → A: モーダルダイアログ（進捗バー＋キャンセルボタン、アプリ操作はブロック）
- Q: インストーラーのダウンロード失敗時の挙動は？ → A: エラーメッセージを表示し「再試行」ボタンで手動リトライ可能、またはキャンセルしてアプリに戻る
- Q: インストーラーのダウンロード保存先は？ → A: OS の一時フォルダ（自動クリーンアップ、ユーザーには見えない）
- Q: 「起動時チェック ON/OFF」設定の操作場所は？ → A: ヘルプメニューにチェック付きメニュー項目「起動時に更新を確認する」を追加
