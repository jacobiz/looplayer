# Quickstart: 自動更新機能 (010-auto-update)

## 手動検証手順

このドキュメントはコードを書かずに機能を手動で確認するための手順です。

---

### US1: 起動時に新バージョンを通知する

**準備**:
- アプリを通常起動する
- GitHub Releases に現在のバージョンより新しいリリースが存在すること

**手順**:
1. `python main.py` でアプリを起動する
2. メインウィンドウが表示された後、数秒以内に更新通知ダイアログが現れることを確認
3. ダイアログに「バージョン X.Y.Z が利用可能です」という文言があることを確認
4. 「あとで」ボタンをクリック → ダイアログが閉じ、アプリが通常動作することを確認
5. アプリを再起動 → 再度ダイアログが表示されることを確認（セッション跨ぎで抑制しない）

**最新の場合の確認**:
- 最新バージョンを使用中の場合、何も表示されないことを確認

**ネットワーク障害時の確認**:
- ネットワーク接続を切断した状態で起動 → ダイアログなしで正常起動することを確認

---

### US2: 手動で更新確認を実行する

**手順**:
1. アプリが起動している状態で、メニュー「ヘルプ」を開く
2. 「更新を確認...」を選択する
3. 最新バージョン使用中の場合: 「最新バージョンです」ダイアログが表示されることを確認
4. 新バージョンがある場合: US1 と同様の更新通知ダイアログが表示されることを確認

**ネットワーク障害時**:
1. ネットワークを切断した状態でヘルプ → 「更新を確認...」を選択
2. エラーダイアログ「更新確認に失敗しました。インターネット接続を確認してください。」が表示されることを確認

---

### US3: 起動時チェックの有効/無効を設定する

**無効化の確認**:
1. ヘルプメニューを開き、「起動時に更新を確認する」のチェックが ON であることを確認
2. 項目をクリックしてチェックを OFF にする
3. アプリを再起動する
4. 更新通知が一切表示されないことを確認
5. `~/.looplayer/settings.json` を開き、`"check_update_on_startup": false` が保存されていることを確認

**再有効化の確認**:
1. ヘルプメニュー → 「起動時に更新を確認する」を再度チェック ON にする
2. アプリを再起動 → 更新チェックが再開されることを確認

---

### ダウンロード進捗ダイアログの確認

**手順**:
1. 新バージョンがある状態でアプリを起動し、更新通知ダイアログを表示させる
2. 「今すぐダウンロード」をクリック
3. 進捗バー付きのモーダルダイアログが表示されることを確認
4. ダウンロード中に「キャンセル」をクリック → ダウンロードが中止され、アプリに戻ることを確認

---

## デバッグ用の設定

開発中にバージョン確認をテストする場合は、`looplayer/updater.py` の `CURRENT_VERSION` を古いバージョン（例: `"0.0.1"`）に一時的に変更することで更新通知を強制的に表示できます。
