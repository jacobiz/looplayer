# Feature Specification: プレイヤー UI バグ修正・操作性改善

**Feature Branch**: `013-player-ui-fixes`
**Created**: 2026-03-18
**Status**: Draft
**Input**: User description: "バグ修正 ・フルスクリーン後に esc キーで解除ができない ・AB点の設定中にシークバー上で位置がわかるようにしたい ・AB点をシークバー上で左右に操作したい ・"ポーズ" "繰返" のインターフェースが潰れて数値が読めないので広く"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - ESC キーでフルスクリーン解除 (Priority: P1)

ユーザーがフルスクリーン表示中に ESC キーを押すと、通常ウィンドウ表示に戻る。現在はキーが効かないバグがあるため、動画視聴中の操作フローが阻害されている。

**Why this priority**: フルスクリーン解除ができないと、ウィンドウモードに戻る方法が限られ、使用不能に近い状態になる重大なバグ。

**Independent Test**: フルスクリーンに切り替え後、ESC キーを押すと通常ウィンドウに戻ることを確認できる。

**Acceptance Scenarios**:

1. **Given** フルスクリーン表示中、**When** ESC キーを押す、**Then** 通常ウィンドウ表示に戻る
2. **Given** フルスクリーン表示中に再生コントロール等のウィジェットがフォーカスを持っている状態、**When** ESC キーを押す、**Then** フォーカス状態に関わらずフルスクリーンが解除される
3. **Given** 通常ウィンドウ表示中、**When** ESC キーを押す、**Then** 何も起きない（通常ウィンドウのまま）

---

### User Story 2 - AB 点のシークバー上リアルタイム表示 (Priority: P2)

ユーザーが A 点または B 点を設定している途中（まだ保存前）でも、現在設定中の A/B 点の位置がシークバー上で視覚的に確認できる。

**Why this priority**: A/B 点の位置を視覚的に確認しながら設定できることで、ループ区間の精度が大幅に向上する。

**Independent Test**: A 点をセットした後、B 点をセットする前の状態でシークバー上に A 点のマーカーが表示されていることを確認できる。

**Acceptance Scenarios**:

1. **Given** 動画が開いている状態、**When** A 点をセットする、**Then** A 点の位置がシークバー上にマーカーとして表示される
2. **Given** A 点がセット済みの状態、**When** B 点をセットする、**Then** A〜B 間の区間がシークバー上にバーとして表示される
3. **Given** A 点のみセット済みの状態（B 点未設定）、**When** シークバーを見る、**Then** A 点の位置に縦線またはマーカーが表示される
4. **Given** AB 点がともに設定されている状態、**When** AB ループをリセットする、**Then** シークバー上のマーカー／バーも消える

---

### User Story 3 - シークバー上で AB 点をドラッグ操作 (Priority: P2)

ユーザーがシークバー上に表示されている A 点または B 点のマーカーをドラッグすることで、A/B 点の位置を直感的に調整できる。

**Why this priority**: ボタン操作やキーボードショートカットより直感的に AB 区間を調整でき、練習・学習用途でのワークフローを大幅に改善する。

**Independent Test**: シークバー上の A 点マーカーをドラッグして新しい位置でリリースすると、A 点が新しい位置に更新されることを確認できる。

**Acceptance Scenarios**:

1. **Given** A 点がシークバー上に表示されている状態、**When** A 点マーカーをドラッグして左右に移動しリリースする、**Then** A 点が新しい位置に更新される
2. **Given** B 点がシークバー上に表示されている状態、**When** B 点マーカーをドラッグして左右に移動しリリースする、**Then** B 点が新しい位置に更新される
3. **Given** A 点マーカーをドラッグ中、**When** B 点より右に移動しようとする、**Then** A 点は B 点位置を超えない
4. **Given** B 点マーカーをドラッグ中、**When** A 点より左に移動しようとする、**Then** B 点は A 点位置を下回らない
5. **Given** AB 点マーカーのドラッグ操作、**When** マウスをリリースする、**Then** A/B 点がドラッグ終点の位置に更新される

---

### User Story 4 - "ポーズ"・"繰返" 入力欄の表示改善 (Priority: P3)

ブックマーク行に表示されている「ポーズ間隔」と「繰返し回数」のスピンボックス・ラベルが狭くて数値が読めない問題を解消し、快適に操作できる幅に改善する。

**Why this priority**: 数値が見えないと誤設定につながる UI バグに近い問題。ただし機能そのものは動作しているため P3。

**Independent Test**: ブックマーク行の「ポーズ」「繰返」スピンボックスに表示された数値（例: 2.5 秒、3 回）が視認可能であることを確認できる。

**Acceptance Scenarios**:

1. **Given** ブックマークが 1 件以上登録されている状態、**When** ブックマーク行を表示する、**Then** 「繰返し回数」スピンボックスの数値（1〜99）が切り取られずに表示される
2. **Given** ポーズ間隔が 0.0 以外の値のブックマーク、**When** ブックマーク行を表示する、**Then** 「ポーズ間隔」スピンボックスの数値（例: 2.5）が切り取られずに表示される
3. **Given** ブックマーク行が表示されている状態、**When** ウィンドウ幅を標準的なサイズに設定する、**Then** すべてのラベルとスピンボックスが重ならず読み取れる

---

### Edge Cases

- フルスクリーン中に SpinBox など編集可能ウィジェットがキーフォーカスを持つとき、ESC キーがウィジェット内で消費されずにフルスクリーン解除に届くか
- AB 点のどちらかが動画の先頭（0ms）または末尾に設定されている場合のドラッグ境界処理
- 動画を読み込んでいない（未開封）状態で AB 点マーカーが残らないか
- ドラッグ操作中に保存済みブックマークのバーが重なっている場合のマーカー選択判定
- A 点のみ設定、B 点のみ設定（通常の操作順序は A→B だが）した場合のシークバー表示

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: ESC キーは、フルスクリーン表示中に押された場合、フォーカスを持つウィジェットの種別に関わらずフルスクリーンを解除しなければならない
- **FR-002**: 通常ウィンドウ表示中に ESC キーを押しても、何も起きてはならない
- **FR-003**: A 点または B 点が設定されている状態では、その位置をシークバー上にリアルタイムで視覚表示しなければならない
- **FR-004**: A 点のみ設定済みの場合、A 点位置に縦線またはマーカーを表示しなければならない；A・B 両方設定済みの場合、A〜B 間を半透明バーで表示しなければならない
- **FR-005**: AB ループリセット時、シークバー上の A/B 点マーカーも同時に消えなければならない
- **FR-006**: シークバー上に表示された A 点マーカーはドラッグ操作で位置を変更でき、変更後の値が一時的な A 点として反映されなければならない（保存済みブックマークのデータは変更しない）
- **FR-007**: シークバー上に表示された B 点マーカーはドラッグ操作で位置を変更でき、変更後の値が一時的な B 点として反映されなければならない（保存済みブックマークのデータは変更しない）
- **FR-008**: A 点ドラッグ中は A 点が B 点位置を超えてはならず、B 点ドラッグ中は B 点が A 点位置を下回ってはならない
- **FR-009**: 「繰返し回数」スピンボックスは、1〜99 の数値が切り取られずに表示できる幅を持たなければならない
- **FR-010**: 「ポーズ間隔」スピンボックスは、0.0〜10.0（小数点以下 1 桁）の数値が切り取られずに表示できる幅を持たなければならない

### Key Entities

- **AB 点 (ab_point_a / ab_point_b)**: 現在設定中の一時的なループ区間の開始・終了位置（ミリ秒）。保存前の一時状態として管理される。
- **BookmarkSlider**: シークバーウィジェット。保存済みブックマーク区間の重ね描きと、クリック/ドラッグによるシーク操作を担う。設定中 AB 点の表示とドラッグ操作も担う対象。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: フルスクリーン中に ESC キーを押すと、いかなるフォーカス状態でも 100% の確率でフルスクリーンが解除される
- **SC-002**: A/B 点を設定した直後（ボタン押下またはショートカット実行後）、シークバー上のマーカーが即座に更新される（体感できる遅延なし）
- **SC-003**: シークバー上の A/B 点マーカーをドラッグして離した後、設定値が正しい位置（ドラッグ終点）に更新される
- **SC-004**: ブックマーク行の「ポーズ」「繰返」スピンボックスで、設定可能な全数値範囲（繰返し: 1〜99、ポーズ: 0.0〜10.0）が視認可能な状態で表示される

## Clarifications

### Session 2026-03-18

- Q: ドラッグ操作は保存済みブックマークを更新するか、一時的な AB 点のみ更新するか？ → A: 一時的な AB 点のみ更新。保存済みブックマークは変更しない。

## Assumptions

- ドラッグ操作中のリアルタイムプレビュー再生は実装しない（ドラッグ終了時に A/B 点を更新するのみ）
- A/B 点マーカーの視覚スタイル（色・形状）は既存のブックマークバーと区別できるデザインとする
- スピンボックスの幅拡張は、行レイアウトが既存のウィンドウ最小幅に収まる範囲で行う
- ESC キーのバグ原因は、SpinBox 等の編集可能ウィジェットがフォーカスを持つ際に QAction のショートカットがブロックされている可能性が高いと想定する
