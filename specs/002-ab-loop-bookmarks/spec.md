# Feature Specification: AB Loop Bookmarks & Sequential Playback

**Feature Branch**: `002-ab-loop-bookmarks`
**Created**: 2026-03-15
**Status**: Draft
**Input**: User description: "ABループポイントをブックマークして複数のループポイントを即座に呼び出して切り替えたい。また、ループポイントの一覧から連続再生(各AB間をつなげて再生)"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - ABループポイントの登録と即時切り替え (Priority: P1)

ユーザーは動画を再生中に任意のA点とB点を設定し、そのループ区間を名前付きでブックマークとして保存できる。保存したブックマークはリスト表示され、クリック一つで即座に切り替えて再生できる。

**Why this priority**: ループポイントのブックマーク機能はこの機能の核心であり、他のすべてのユーザーストーリーの前提となる。この機能単独でも「複数箇所の繰り返し練習」という主要ニーズを満たす。

**Independent Test**: ブックマーク登録→一覧表示→切り替え再生の一連の操作をテストすることで、それだけで独立した価値を検証できる。

**Acceptance Scenarios**:

1. **Given** 動画が再生中で現在位置がある、**When** ユーザーがA点を設定後B点を設定してブックマーク保存を実行する、**Then** ループ区間がブックマーク一覧に追加され、設定したA〜B間でループ再生が開始される
2. **Given** 複数のブックマークが一覧に存在する、**When** ユーザーが一覧から別のブックマークを選択する、**Then** 即座にその区間のA点に移動しループ再生に切り替わる（現在再生を中断して）
3. **Given** ブックマークが存在する、**When** ユーザーがブックマーク名を編集する、**Then** 一覧の表示名が更新され、ループ区間（A・B点）は変わらない
4. **Given** ブックマークが一覧に存在する、**When** ユーザーがそのブックマークを削除する、**Then** 一覧から削除され、削除前に再生中だった場合はループが解除される

---

### User Story 2 - ブックマーク一覧からの連続再生 (Priority: P2)

ユーザーはブックマーク一覧に登録された複数のAB区間を順番につなげて連続再生できる。各区間が終わると自動的に次の区間へ移動し、最後の区間の後はリストの先頭に戻って継続する（ユーザーが停止するまで）。

**Why this priority**: 連続再生は複数ブックマークの存在を前提とするため P2。語学学習や楽器練習などで「複数の練習箇所をまとめて通し練習する」ニーズに応える。

**Independent Test**: 複数のブックマークを登録し、連続再生モードを起動して各区間が順番に再生されるかを独立して検証できる。

**Acceptance Scenarios**:

1. **Given** 複数のブックマークが登録されている、**When** ユーザーが連続再生を開始する、**Then** リストの先頭ブックマークから順に各AB区間を再生し、1区間終了後は自動的に次の区間へ移動する
2. **Given** 連続再生中、**When** 最後のブックマーク区間の再生が終了する、**Then** リストの先頭に戻って繰り返す（ユーザーが停止するまで継続）
3. **Given** 連続再生が進行中、**When** ユーザーが連続再生を停止する、**Then** 現在の区間の再生を止め、通常再生モード（ループなし）に戻る
4. **Given** 連続再生中で現在の区間を何度か繰り返したい、**When** ユーザーが各区間の繰り返し回数を設定する、**Then** 指定回数分その区間を繰り返してから次の区間へ進む

---

### User Story 3 - ブックマークの永続化と管理 (Priority: P3)

ユーザーが保存したブックマークは動画ファイルに紐づけて保存され、次回同じ動画を開いたときに自動的に読み込まれる。

**Why this priority**: 永続化がなくても一時的な使用は可能なため P3。しかし学習・練習用途では繰り返し利用するため、長期的な価値提供に重要。

**Independent Test**: ブックマーク保存→アプリ再起動→同じ動画を開く→ブックマーク一覧が復元されるかを独立して検証できる。

**Acceptance Scenarios**:

1. **Given** 動画にブックマークが登録されている、**When** アプリを終了して再起動し同じ動画を開く、**Then** 以前登録したブックマーク一覧がすべて復元される
2. **Given** 複数の動画それぞれにブックマークがある、**When** 異なる動画を開く、**Then** その動画に対応するブックマークのみが表示される（他の動画のブックマークは表示されない）
3. **Given** ブックマークを並び替えたい、**When** ユーザーが一覧のブックマークをドラッグ＆ドロップで並び替える、**Then** 並び順が更新され、連続再生時もその順番に従う

---

### Edge Cases

- A点とB点が同じ位置に設定された場合、またはB点がA点より前にある場合はどう処理するか？（警告表示して保存を阻止）
- 動画の総時間より長い位置にB点が設定された場合の挙動は？（動画終端をB点として扱うか警告）
- ブックマークが1つしかない状態で連続再生を開始した場合は？（単独ループとして動作）
- 動画ファイルを移動・削除した場合、紐づいたブックマークデータはどう扱うか？（ファイルパスをキーとするため、移動・リネームするとブックマークは孤立し復元不可。データ自体はストレージに残る）
- ブックマークが0件の状態で連続再生ボタンを押した場合は？（ボタンを無効化またはメッセージ表示）

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: ユーザーはA点・B点が設定済みの状態で専用「ブックマーク保存」ボタンを押すことで、現在のAB区間を名前付きブックマークとして保存できなければならない（A点またはB点が未設定の場合はボタンを無効化する）
- **FR-002**: システムはブックマーク一覧をパネルまたはサイドバーに表示しなければならない
- **FR-003**: ユーザーはブックマーク一覧から任意のブックマークを選択し、即座にそのAB区間のループ再生に切り替えられなければならない
- **FR-004**: ユーザーはブックマーク名をインラインで編集できなければならない
- **FR-005**: ユーザーはブックマークを削除できなければならない（確認ダイアログなしで即時削除）
- **FR-006**: ユーザーは連続再生モードを起動でき、ブックマーク一覧のAB区間を登録順に順番に再生できなければならない
- **FR-007**: 連続再生中は現在再生中の区間名と次の区間名がUIに表示されなければならない
- **FR-008**: ブックマークデータは動画ファイルパスに紐づいてアプリのローカル設定として永続化され、次回起動時に自動で読み込まれなければならない
- **FR-009**: ユーザーはブックマーク一覧の並び順をドラッグ＆ドロップで変更できなければならない
- **FR-010**: ブックマーク一覧の各行に繰り返し回数をインライン表示し、ユーザーはいつでも変更できなければならない（デフォルト: 1回、連続再生時に適用）
- **FR-011**: A点がB点以降にある場合、またはいずれかのポイントが動画長を超える場合、システムはユーザーに警告を表示しブックマークの保存を防止しなければならない

### Key Entities

- **LoopBookmark（ループブックマーク）**: 1つのAB区間を表す。属性: 一意ID、動画ファイルパス（紐付け用）、A点タイムスタンプ（ミリ秒）、B点タイムスタンプ（ミリ秒）、表示名、並び順インデックス、繰り返し回数
- **BookmarkList（ブックマーク一覧）**: 特定動画に紐づくLoopBookmarkの順序付きコレクション。動画ファイルの絶対パスをキーとして管理（ファイル移動・リネーム時はブックマークが孤立し復元不可）
- **SequentialPlaySession（連続再生セッション）**: 連続再生の進行状態を表す一時的なオブジェクト。属性: 現在の区間インデックス、現在区間の残り繰り返し回数、再生中フラグ

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーは現在再生中の位置からAB区間をブックマークとして3ステップ以内の操作で保存できる
- **SC-002**: ブックマーク一覧からの切り替え操作後、0.5秒以内に指定区間のループ再生が開始される
- **SC-003**: ブックマーク一覧に20件以上登録されている場合でも、切り替え操作は0.5秒以内に完了する（SC-002 と同基準）
- **SC-004**: アプリ再起動後、前回保存したブックマークが100%の再現率（タイムスタンプ・名前・並び順を含む）で復元される
- **SC-005**: 連続再生モードで、各区間の終了から次の区間開始までの遷移が0.5秒以内に完了する

## Clarifications

### Session 2026-03-15

- Q: 動画ファイルの識別キーは何を使うか？ → A: ファイルパスのみ（Option A）。移動・リネームした場合はブックマークが孤立し復元不可とする。
- Q: ブックマーク保存のトリガー操作は何か？ → A: 専用「ブックマーク保存」ボタン（A点・B点が設定済み時のみ有効化）。
- Q: 繰り返し回数はいつ・どこで設定するか？ → A: ブックマーク一覧の各行にインライン表示・編集（保存後いつでも変更可能）。
- Q: ブックマークのエクスポート/インポートはスコープに含めるか？ → A: 今回スコープ外（ローカル保存のみ）。

### Out of Scope

- ブックマークデータのエクスポート/インポート（ファイル共有・バックアップ機能）は本機能のスコープ外とする

## Assumptions

- 既存のABループ機能（A点・B点の設定・ループ再生）がすでに動作しており、本機能はその上にブックマーク管理と連続再生を追加する
- ブックマークの上限数は100件とする（UIおよびパフォーマンスの観点から妥当な上限）
- ブックマークデータの保存はアプリのローカルストレージ（設定ファイルやDBなど）とし、動画ファイル自体を変更しない
- 連続再生は最後の区間終了後にリスト先頭に戻って継続する（デフォルト動作）
- ブックマーク名が未入力の場合、「ブックマーク N」（Nは登録順番号）をデフォルト名とする
- 繰り返し回数のデフォルト値は1回（各区間を1回ずつ再生）とする
