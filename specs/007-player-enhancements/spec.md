# Feature Specification: プレイヤー機能強化

**Feature Branch**: `007-player-enhancements`
**Created**: 2026-03-16
**Status**: Draft
**Input**: 優先度高・中の機能をすべて追加

## 概要

語学学習・楽器練習・動画分析に特化した LoopPlayer に、現在不足している再生操作の精細化・マルチトラック対応・学習補助機能を追加する。既存のABループ・ブックマーク機能を補完し、より実用的な練習ツールとする。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 精細な再生操作 (Priority: P1)

語学学習者や楽器練習者が、発音・演奏の細部を確認するために、1フレーム単位の送り操作・1秒単位のシーク・キーボードだけで速度を即座に変更できる。

**Why this priority**: 既存の ±5秒 シークと速度変更（メニュー操作のみ）では細部確認の操作が煩雑。フレームコマ送りは ABループ精度改善の補完的手段でもある。実装コストが低く効果が大きい。

**Independent Test**: 動画を開いてキーボードのみで「コマ送り→±1秒シーク→速度変更」を実施できれば独立してテスト可能。

**Acceptance Scenarios**:

1. **Given** 動画が一時停止中, **When** `.` キーを押す, **Then** 映像が1フレーム進み、一時停止状態を維持する
2. **Given** 動画が一時停止中, **When** `,` キーを押す, **Then** 映像が1フレーム戻る（可能な限り）
3. **Given** 動画が再生中, **When** `.` または `,` キーを押す, **Then** 自動で一時停止し、その位置から1フレーム進める／戻す
3. **Given** 動画が再生中または一時停止中, **When** `Shift+→` を押す, **Then** 再生位置が1秒進む
4. **Given** 動画が再生中または一時停止中, **When** `Shift+←` を押す, **Then** 再生位置が1秒戻る
5. **Given** 動画が再生中または一時停止中, **When** `Ctrl+→` を押す, **Then** 再生位置が10秒進む
6. **Given** 動画が再生中または一時停止中, **When** `Ctrl+←` を押す, **Then** 再生位置が10秒戻る
7. **Given** 再生速度が1.0倍, **When** `]` キーを押す, **Then** 速度が次の段階（1.25倍）に上がる
8. **Given** 再生速度が1.0倍, **When** `[` キーを押す, **Then** 速度が前の段階（0.75倍）に下がる
9. **Given** 再生速度が最大値（2.0倍）, **When** `]` を押す, **Then** 速度は変わらず、ステータスに通知が出る
10. **Given** 再生速度が最小値（0.5倍）, **When** `[` を押す, **Then** 速度は変わらず、ステータスに通知が出る

---

### User Story 2 - マルチトラック対応（音声・字幕） (Priority: P2)

日英二言語収録の教材動画や字幕付きコンテンツを扱う学習者が、メニューから音声トラックや字幕トラックを切り替えられる。

**Why this priority**: 語学学習教材で最も頻繁に使われる機能の一つ。実装コストが低い。

**Independent Test**: 複数の音声トラックを持つ MKV ファイルを開き、メニューからトラックを切り替えられれば独立してテスト可能。

**Acceptance Scenarios**:

1. **Given** 複数音声トラックを持つ動画を開いた状態, **When** 「再生 → 音声トラック」メニューを開く, **Then** 利用可能なトラックの一覧が表示される（言語名またはトラック番号）
2. **Given** 音声トラック一覧が表示された状態, **When** 別のトラックを選択する, **Then** 即座に選択したトラックで再生される
3. **Given** 字幕トラックを持つ動画を開いた状態, **When** 「再生 → 字幕」メニューを開く, **Then** 利用可能な字幕トラックの一覧と「字幕なし」オプションが表示される
4. **Given** 字幕トラック一覧が表示された状態, **When** 字幕トラックを選択する, **Then** 映像上に字幕が表示される
5. **Given** 字幕が表示されている状態, **When** 「字幕なし」を選択する, **Then** 字幕が非表示になる
6. **Given** 音声・字幕トラックが1つしかない動画を開いた状態, **When** 対応メニューを開く, **Then** メニュー項目がグレーアウトされている
7. **Given** 別の動画ファイルを開いた, **When** 新しい動画を読み込む, **Then** 音声・字幕トラックの選択が新しい動画の情報にリセットされる

---

### User Story 3 - スクリーンショット保存 (Priority: P3)

語学学習者や分析者が、参考にしたい動画の特定フレームを静止画として保存できる。

**Why this priority**: 教材の重要場面をメモとして残す、テロップを保存するなど学習補助用途に有用。

**Independent Test**: 動画を一時停止してスクリーンショット操作を実行し、デスクトップにファイルが保存されることを確認できれば独立してテスト可能。

**Acceptance Scenarios**:

1. **Given** 動画が再生中または一時停止中, **When** `Ctrl+Shift+S` を押す（またはメニュー「ファイル → スクリーンショット」）, **Then** 現在のフレームが PNG 形式でデスクトップに保存される
2. **Given** スクリーンショットを実行した, **When** 保存が完了する, **Then** 保存先のフルパスを示す通知がステータスバーに数秒間表示される
3. **Given** 連続でスクリーンショットを実行した, **When** 複数回保存する, **Then** タイムスタンプ（YYYYMMDD_HHMMSS）を含むファイル名で上書きされずに保存される
4. **Given** 動画を開いていない状態, **When** スクリーンショット操作を実行する, **Then** メニューがグレーアウトされており操作できない

---

### User Story 4 - 再生終了時の自動動作 (Priority: P4)

ユーザーが、動画が終端まで再生されたときの動作（停止・先頭に戻る・自動ループ）をあらかじめ設定しておける。

**Why this priority**: 現状は終端で止まったまま何も起きないため、繰り返し視聴・ながら学習の妨げになっている。

**Independent Test**: 「再生終了後にループ」を設定して動画を再生し、終端で先頭から自動再生されることを確認できれば独立してテスト可能。

**Acceptance Scenarios**:

1. **Given** 「再生終了時: 停止（デフォルト）」の設定, **When** 動画が終端に達する, **Then** 再生が停止し、シークバーが末尾に留まる
2. **Given** 「再生終了時: 先頭に戻る」の設定, **When** 動画が終端に達する, **Then** 再生が停止し、シークバーが先頭に戻る
3. **Given** 「再生終了時: ループ再生」の設定, **When** 動画が終端に達する, **Then** 先頭から自動的に再生が再開される
4. **Given** 設定が変更された, **When** 次に動画が終端に達する, **Then** 新しい設定が即座に適用される
5. **Given** ABループが有効な状態, **When** 動画が終端に達する, **Then** ABループの制御が優先され、再生終了時の設定は適用されない
6. **Given** 連続再生（ブックマーク順再生）が有効な状態, **When** 最後のブックマーク区間が終了する, **Then** 連続再生の制御が優先される

---

### User Story 5 - 再生位置の記憶と復元 (Priority: P5)

同じ動画ファイルを繰り返し開く学習者が、前回閉じたときの再生位置から自動的に再開できる。

**Why this priority**: 長時間の教材（講義動画・映画）を複数セッションに分けて学習するときに毎回スクロールする手間をなくす。

**Independent Test**: 動画を途中まで再生して閉じ、同じファイルを再度開くと前回位置から始まることを確認できれば独立してテスト可能。

**Acceptance Scenarios**:

1. **Given** 動画を途中まで再生した状態, **When** 別のファイルを開くか、アプリを終了する, **Then** その時点の再生位置が保存される
2. **Given** 前回の再生位置が保存された動画, **When** 同じファイルをアプリで開く, **Then** 保存された再生位置から一時停止状態で再開する
3. **Given** 動画の95%以上まで再生した状態で閉じた, **When** 同じファイルを開く, **Then** 先頭から始まる（「見終わった」とみなしてリセット）
4. **Given** 動画の開始から5秒未満しか再生していない状態で閉じた, **When** 同じファイルを開く, **Then** 先頭から始まる（位置を記憶しない）

---

### User Story 6 - ブックマークへのメモ追加 (Priority: P6)

学習者が、ブックマーク（ABループ区間）にテキストメモを付けて、その区間のポイントや覚書を残せる。

**Why this priority**: 「このフレーズは〜」「ここの発音に注意」などの情報をアプリ内に保持することで、外部メモアプリとの行き来をなくす。

**Independent Test**: ブックマーク行にメモアイコンを追加し、メモを入力・保存・再表示できれば独立してテスト可能。

**Acceptance Scenarios**:

1. **Given** ブックマーク一覧が表示されている, **When** ブックマーク行のメモアイコンをクリックする, **Then** メモ入力ダイアログが開く
2. **Given** メモ入力ダイアログが開いている, **When** テキストを入力して保存する, **Then** メモがブックマークに紐付いて保存され、メモアイコンが「メモあり」状態に変わる
3. **Given** メモが保存されているブックマーク, **When** メモアイコンをクリックする, **Then** 保存済みのメモ内容を含むダイアログが開き編集できる
4. **Given** メモが保存されているブックマーク, **When** メモを空にして保存する, **Then** 「メモあり」状態が解除される
5. **Given** ブックマークをエクスポートした, **When** JSON ファイルを確認する, **Then** メモの内容がエクスポートデータに含まれる
6. **Given** メモ付きブックマークを含む JSON をインポートした, **When** インポートが完了する, **Then** メモが正しく復元されている

---

### User Story 7 - フォルダドロップでプレイリスト (Priority: P7)

学習者が、動画ファイルを含むフォルダをウィンドウにドロップするだけで、フォルダ内の動画を順番に再生できる。

**Why this priority**: 連続した講義動画・シリーズ動画を個別に開く手間をなくす。

**Independent Test**: 動画ファイルを含むフォルダをドロップし、最初のファイルが再生され終了後に次のファイルに進めば独立してテスト可能。

**Acceptance Scenarios**:

1. **Given** 動画ファイルを含むフォルダをアプリにドロップした, **When** ドロップが完了する, **Then** フォルダ内の対応フォーマットの動画がファイル名順にプレイリストとして読み込まれ、最初のファイルが再生される
2. **Given** プレイリストが読み込まれた状態, **When** 現在の動画が終了する, **Then** 自動的に次の動画に進む
3. **Given** プレイリストの最後の動画が終了した, **When** 動画が終端に達する, **Then** 再生が停止する
4. **Given** プレイリストが読み込まれた状態, **When** 別の単一ファイルを開く, **Then** プレイリストが解除され、開いたファイルのみの再生に切り替わる
5. **Given** 動画ファイルを含まないフォルダをドロップした, **When** ドロップが完了する, **Then** 「対応する動画ファイルが見つかりませんでした」というメッセージが表示される
6. **Given** フォルダ内に対応動画が1ファイルのみの場合, **When** ドロップが完了する, **Then** そのファイルを単体で開く

---

### Edge Cases

- **コマ送り（戻り）**: 動画フォーマットによっては1フレーム前への移動が正確でない場合がある（プレイヤーの仕様による）。その場合、可能な範囲で近いフレームに戻る
- **音声・字幕トラック**: 動画読み込み直後はトラック情報が未取得の場合がある。メニューを開いたタイミングで最新情報を取得する
- **スクリーンショット保存失敗**: デスクトップへの書き込み権限がない場合、エラーメッセージを表示してユーザーに通知する
- **再生位置の記憶**: 保存件数の上限は10件とし、`positions.json` として独立管理する（最近開いたファイルとは別ストレージ）。10件を超えた分は古い順に削除する
- **フォルダドロップ**: サブフォルダ内の動画は対象外（1階層のみ）。隠しファイル（`.` 始まり）は除外する
- **速度ショートカットの端**: 最大・最小速度で `]`/`[` を押した場合、ステータスバーに「最大速度です」「最小速度です」を表示する

## Requirements *(mandatory)*

### Functional Requirements

**精細な再生操作（US1）**

- **FR-001**: ユーザーは `.` キーで映像を1フレーム進めることができる。再生中に押した場合は自動で一時停止してからフレームを進める
- **FR-002**: ユーザーは `,` キーで映像を1フレーム戻すことができる。再生中に押した場合は自動で一時停止してからフレームを戻す
- **FR-003**: ユーザーは `Shift+→` / `Shift+←` で ±1秒シークができる
- **FR-004**: ユーザーは `Ctrl+→` / `Ctrl+←` で ±10秒シークができる
- **FR-005**: ユーザーは `]` キーで再生速度を1段階上げることができる（既存6段階と同一）
- **FR-006**: ユーザーは `[` キーで再生速度を1段階下げることができる（既存6段階と同一）
- **FR-007**: 速度が最大または最小の端に達している場合、それ以上の方向には変化せずステータスバーに2秒間通知を表示する

**マルチトラック対応（US2）**

- **FR-008**: 「再生 → 音声トラック」サブメニューに、現在の動画の音声トラック一覧を表示する
- **FR-009**: 音声トラックを選択すると即座に切り替わる
- **FR-010**: 「再生 → 字幕」サブメニューに、字幕トラック一覧と「字幕なし」オプションを表示する
- **FR-011**: 字幕トラックを選択すると映像上に字幕が表示される。「字幕なし」で非表示になる
- **FR-012**: 新しい動画を開くたびに音声・字幕のサブメニューを更新する
- **FR-013**: トラックが1種類しかない動画ではサブメニュー項目をグレーアウトする

**スクリーンショット（US3）**

- **FR-014**: `Ctrl+Shift+S` またはメニュー「ファイル → スクリーンショット」で現在フレームをデスクトップに PNG 保存する
- **FR-015**: ファイル名は `LoopPlayer_YYYYMMDD_HHMMSS.png` 形式とする
- **FR-016**: 保存完了後、保存先パスをステータスバーに3秒間表示する
- **FR-017**: 動画を開いていない状態ではスクリーンショットのメニュー項目がグレーアウトされる

**再生終了時の動作（US4）**

- **FR-018**: 「再生 → 再生終了時の動作」メニューに「停止」「先頭に戻る」「ループ再生」の3択を排他的に提供する。デフォルトは「停止」。選択した設定はアプリ再起動後も維持される
- **FR-019**: 設定に応じて動画終端到達時の動作が変わる
- **FR-020**: ABループ有効中・連続再生中は再生終了時の設定より各機能の制御が優先される

**再生位置の記憶（US5）**

- **FR-021**: 別ファイルを開く・アプリを終了する際、現在の再生位置をファイルパスに紐付けて保存する
- **FR-022**: 同じファイルを開いたとき、保存済みの再生位置がある場合はその位置から一時停止状態で再開する
- **FR-023**: 動画の再生完了位置が全体の95%以上の場合、または開始から5秒未満の場合は再生位置を保存しない（先頭から再開）

**ブックマークメモ（US6）**

- **FR-024**: 各ブックマーク行にメモアイコン（ボタン）を追加する。メモが存在する場合とない場合でアイコンの外観を区別する
- **FR-025**: メモアイコンをクリックするとテキスト入力ダイアログが開き、メモを入力・確認・編集できる
- **FR-026**: メモはブックマークデータ（JSON ファイル）に保存・復元される
- **FR-027**: エクスポート/インポートの JSON にメモフィールドを含める

**フォルダドロップ（US7）**

- **FR-028**: アプリへのフォルダドロップを認識し、フォルダ内の動画ファイル（対応フォーマット、1階層のみ、隠しファイル除外）をファイル名順でプレイリストとして読み込む
- **FR-029**: プレイリスト再生中は現在のファイルが終了すると自動で次のファイルに進む
- **FR-030**: 単一ファイルを開いた場合（ドロップまたはダイアログ）はプレイリストを解除する
- **FR-031**: フォルダ内に対応動画がない場合はエラーメッセージを表示する

### Key Entities

- **PlaybackPosition**: 動画ファイルパスに紐づいた再生位置（ミリ秒）。最大10件を保持し古い順に削除
- **Playlist**: フォルダドロップで生成される動画ファイルのリスト。ファイルパスの配列と現在インデックスを持つ。セッション中のみ保持（永続化しない）
- **BookmarkNote**: ブックマークに付属するテキストメモ。`LoopBookmark` の `notes` フィールドとして既存データに追加

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: キーボードのみで「フレームコマ送り・精細シーク・速度変更」の3操作すべてが完結できる（マウス操作不要）
- **SC-002**: 音声・字幕トラックの切り替えが3ステップ以内（メニュー開く→サブメニュー開く→選択）で完了する
- **SC-003**: スクリーンショットが操作から1秒以内にデスクトップに保存される
- **SC-004**: 再生終了時の動作設定が即座に（次の終端到達から）反映される
- **SC-005**: 前回の再生位置への復元が、ファイルを開いてから3秒以内に完了する
- **SC-006**: ブックマークメモの入力・保存・再表示が5ステップ以内で完結する
- **SC-007**: フォルダドロップ後、プレイリスト最初の動画の再生開始まで3秒以内に完了する
- **SC-008**: 追加した全ショートカットがショートカット一覧ダイアログに正しく記載される

## Clarifications

### Session 2026-03-16

- Q: PlaybackPosition の保存件数管理 → A: `positions.json` として独立管理（最近開いたファイルとは別）。最大10件、古い順に削除
- Q: 再生終了時の動作設定（US4）の永続化 → A: アプリ再起動後も設定を維持する（`~/.looplayer/settings.json` に保存）
- Q: フレームコマ送り（`.`/`,`）を再生中に押した場合の動作 → A: 自動で一時停止し、その位置から1フレーム進める／戻す

## Assumptions

- フレームコマ送り（戻り）の精度は動画フォーマット・コーデックに依存するため、「可能な範囲で1フレーム戻す」動作を仕様とし、完全な逆方向フレーム精度は保証しない
- スクリーンショットの保存先はデスクトップ固定とする（保存先選択UIは対象外）
- 再生終了時の動作設定は `~/.looplayer/settings.json` に保存し、アプリ再起動後も維持する
- フォルダドロップ時のプレイリストはセッション中のみ保持し、アプリ再起動後には復元しない
- 字幕レンダリングはプレイヤーの内部機能で処理されるため、フォント・スタイルのカスタマイズは対象外
- 再生位置の保存件数の上限は10件とし、最近開いたファイル（10件）と別途管理する
- プレイリスト再生中のブックマーク・ABループは現在の動画ファイルに紐づくもののみ適用される
- フォルダ内に対応動画が1ファイルのみの場合はプレイリストとして扱わず単体再生とし、US4 の再生終了時の動作設定が通常通り適用される

## Out of Scope

- 字幕ファイル（SRT/ASS等）の外部読み込み
- プレイリストのGUI表示・並び替え（フォルダドロップ時の自動順再生のみ）
