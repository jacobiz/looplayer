# Feature Specification: コンテキストメニューの充実

**Feature Branch**: `022-enhance-context-menu`
**Created**: 2026-03-22
**Status**: Draft
**Input**: User description: "右クリックメニュー(コンテキストメニュー)を充実させたい"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 動画エリアの右クリックメニュー（新規追加） (Priority: P1)

動画再生エリア（映像フレーム）を右クリックすると、よく使う操作へ素早くアクセスできるコンテキストメニューが表示される。
現在、動画エリアへの右クリックは何も反応しないため、メニューバーまでカーソルを移動せずに操作できるようになる。

**Why this priority**: 動画エリアは最も視線が集まる場所であり、右クリックを習慣的に使うユーザーが最初に試みる場所。再生制御・A/B 点設定などの主要操作への最短経路を提供する。

**Independent Test**: 動画ファイルを開いた状態で動画エリアを右クリックするとメニューが表示され、各項目が機能することで単独でテスト可能。

**Acceptance Scenarios**:

1. **Given** 動画ファイルが開いている, **When** 動画エリアを右クリック, **Then** コンテキストメニューが表示される
2. **Given** コンテキストメニューが表示されている, **When** 「再生 / 一時停止」を選択, **Then** 再生状態が切り替わる
3. **Given** 動画再生中, **When** 動画エリアを右クリックして「A 点を設定」を選択, **Then** 現在位置が A 点として設定される
4. **Given** 動画再生中, **When** 動画エリアを右クリックして「B 点を設定」を選択, **Then** 現在位置が B 点として設定される
5. **Given** A 点と B 点が設定済み, **When** 動画エリアを右クリックして「ここにブックマークを追加」を選択, **Then** 現在の A/B 点でブックマークが作成される
6. **Given** ファイルが開いていない, **When** 動画エリアを右クリック, **Then** 再生操作系メニューは無効（グレーアウト）で表示される

---

### User Story 2 - ブックマーク行の右クリックメニュー拡充 (Priority: P2)

現在ブックマーク行を右クリックすると「クリップを書き出す」と「再生回数をリセット」の2項目しか表示されない。
より多くの操作をコンテキストメニューから実行できるようにすることで、ブックマーク操作の効率を向上させる。

**Why this priority**: ブックマークパネルはすでにコンテキストメニューを持つため改修コストが低く、ユーザーが「コンテキストメニューを充実させたい」と感じやすい場所でもある。

**Independent Test**: ブックマーク行を右クリックして、拡充された項目（ジャンプ・名前変更・削除・複製など）が動作することで単独でテスト可能。

**Acceptance Scenarios**:

1. **Given** ブックマークが1件以上存在する, **When** ブックマーク行を右クリック, **Then** 拡充されたコンテキストメニューが表示される
2. **Given** コンテキストメニューが表示されている, **When** 「A 点へジャンプ」を選択, **Then** 再生位置がそのブックマークの A 点に移動し、ジャンプ前の再生状態（再生中／一時停止）が維持される
3. **Given** コンテキストメニューが表示されている, **When** 「名前を変更」を選択, **Then** 名前入力ダイアログが開き保存できる
4. **Given** コンテキストメニューが表示されている, **When** 「削除」を選択, **Then** そのブックマークが削除される（既存の Undo 仕組みと連動）
5. **Given** コンテキストメニューが表示されている, **When** 「複製」を選択, **Then** 同一の A/B 点・メモ・タグ・名前（末尾に「のコピー」付き）のブックマークが元行の直後に挿入される
6. **Given** A 点と B 点が両方設定済み, **When** 「クリップを書き出す」を選択, **Then** 書き出しダイアログが開く（既存動作）
7. **Given** A 点または B 点が未設定, **When** コンテキストメニューを表示, **Then** 「クリップを書き出す」がグレーアウトされる（既存動作の維持）

### User Story 3 - ブックマークパネル空白エリアの右クリックメニュー（新規追加） (Priority: P3)

ブックマーク行以外の空白エリア（パネルの余白）を右クリックすると、ブックマーク一覧全体に対する操作メニューが表示される。

**Why this priority**: ブックマークの一括操作（インポート・エクスポート等）へのアクセスポイントを増やすことで利便性が上がるが、メニューバーで代替可能なため優先度は低め。

**Independent Test**: ブックマークパネルの空白部分を右クリックしてメニューが表示され、「インポート」「エクスポート」が機能することで単独でテスト可能。

**Acceptance Scenarios**:

1. **Given** ブックマークパネルが表示されている, **When** パネルの空白エリアを右クリック, **Then** パネル操作用コンテキストメニューが表示される
2. **Given** コンテキストメニューが表示されている, **When** 「ブックマークをインポート」を選択, **Then** ファイル選択ダイアログが開く（既存のインポート処理を呼び出す）
3. **Given** ブックマークが1件以上ある, **When** 「ブックマークをエクスポート」を選択, **Then** 保存先選択ダイアログが開く（既存のエクスポート処理を呼び出す）
4. **Given** ブックマークが0件, **When** コンテキストメニューを表示, **Then** 「エクスポート」がグレーアウトされる

---

### Edge Cases

- フルスクリーンモード中でも動画エリアのコンテキストメニューが表示できること
- コンテキストメニュー表示中にファイルがアンロードされた場合でも例外が発生しないこと
- A 点のみ設定・B 点のみ設定の場合、「クリップを書き出す」は無効のまま
- A 点または B 点が未設定の場合、「ここにブックマークを追加」もグレーアウトされる（A/B 両方設定済みのときのみ有効）

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 動画エリアを右クリックしたとき、コンテキストメニューを表示しなければならない
- **FR-002**: 動画エリアのコンテキストメニューには「再生 / 一時停止」「停止」「A 点を設定」「B 点を設定」「ここにブックマークを追加」「スクリーンショット」「フルスクリーン切り替え」を含めなければならない。「ここにブックマークを追加」は A/B 点が両方設定済みの場合のみ有効とし、未設定時はグレーアウトする
- **FR-003**: ファイルが開かれていない場合、動画操作系メニュー項目は無効状態で表示しなければならない
- **FR-004**: ブックマーク行のコンテキストメニューに「A 点へジャンプ」「名前を変更」「複製」「削除」を追加しなければならない。「A 点へジャンプ」はジャンプ前の再生状態（再生中／一時停止）を変えずにシークする
- **FR-005**: ブックマーク行の「削除」はメインの削除ボタンと同じ Undo 可能な削除フローを経由しなければならない
- **FR-006**: ブックマーク行の「複製」は元のブックマークと同一の A/B 点・メモ・タグをコピーし、名前末尾に「のコピー」（英語環境では " Copy"）を付加して元ブックマークの直後に挿入しなければならない
- **FR-007**: ブックマークパネルの空白エリアを右クリックしたとき、「ブックマークをインポート」「ブックマークをエクスポート」を含むコンテキストメニューを表示しなければならない
- **FR-008**: 既存のメニューバー・ショートカットキーと重複する操作は、同一のアクションオブジェクトを再利用しなければならない（動作の一貫性確保）
- **FR-009**: すべてのコンテキストメニュー項目の文言は `i18n.py` の `t()` 経由で取得しなければならない

### Key Entities

- **コンテキストメニュー（動画エリア）**: 再生制御・A/B 点操作・ブックマーク追加・ファイル操作を束ねたメニュー
- **コンテキストメニュー（ブックマーク行）**: 個別ブックマークへの操作（ジャンプ・編集・複製・削除・書き出し）を束ねたメニュー
- **コンテキストメニュー（パネル空白）**: ブックマーク一覧全体への操作（インポート・エクスポート）を束ねたメニュー

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 動画エリアの右クリックから主要操作（再生/一時停止・A/B 点設定）に到達するまでのクリック数が 1 回に収まる
- **SC-002**: ブックマーク行のコンテキストメニューが現在の 2 項目から 6 項目以上に拡充される
- **SC-003**: コンテキストメニュー経由の操作結果が、メニューバー・ショートカット経由と同一の結果になる（整合性 100%）
- **SC-004**: 既存の全テストが引き続きパスする（リグレッションゼロ）

## Clarifications

### Session 2026-03-22

- Q: A/B 点が未設定の場合に「ここにブックマークを追加」はどう振る舞うか → A: A/B 点が両方設定済みのときのみ有効（未設定時はグレーアウト）
- Q: 「A点へジャンプ」後の再生状態はどうするか → A: 現在の再生状態を維持（再生中→そのまま再生、一時停止中→一時停止のまま）
- Q: 「複製」で作成されたブックマークの挿入位置はどこか → A: 元ブックマークの直後に挿入

## Assumptions *(optional)*

- 動画エリアのコンテキストメニューに「速度変更」サブメニューは含めない（階層が深くなるため P1 スコープ外とする）
- ブックマーク行の「名前を変更」は既存のダブルクリック編集と同一の UI を使用する
- フルスクリーン時の動画エリア右クリックは、通常ウィンドウ時と同じメニューを表示する
- 右クリック操作はマウスのみ対象（タッチスクリーンの長押しは対象外）
