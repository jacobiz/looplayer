# Feature Specification: 字幕からのブックマーク自動生成とデータ一括バックアップ

**Feature Branch**: `019-subtitle-bookmark-backup`
**Created**: 2026-03-19
**Status**: Draft
**Input**: F-202 字幕からブックマーク自動生成、F-402 データの一括バックアップ・復元

## Clarifications

### Session 2026-03-19

- Q: 「字幕からブックマーク生成」のメニュー配置はどこか？ → A: 再生（Playback）> 字幕 サブメニュー内（「字幕ファイルを開く...」の下）
- Q: 既存ブックマークと A点・B点が一致する重複エントリの扱いは？ → A: 重複チェックなし・そのまま追記する（ユーザー責任）
- Q: データ復元後の反映タイミングは？ → A: 再起動必須 — 「アプリを再起動してください」とメッセージを表示して終了する

## User Scenarios & Testing *(mandatory)*

### User Story 1 — 字幕ファイルからブックマークを一括生成する (Priority: P1)

映画やドラマで語学学習をしているユーザーが、SRT 字幕ファイルを読み込んだ動画に対して「字幕からブックマーク生成」を実行すると、各字幕のタイムスタンプを A/B 点・字幕テキストをブックマーク名として、すべての字幕区間がブックマークとして一括登録される。

**Why this priority**: 長い動画での手動 A/B 点設定は膨大な手間。SRT ファイルが存在する語学学習コンテンツでは最も利用頻度が高い操作であり、looplay! の練習ツールとしての価値を大きく高める。

**Independent Test**: 字幕ファイルを読み込んだ状態でメニューから「字幕からブックマーク生成」を実行し、ブックマークパネルに字幕区間が登録されることを確認できる。

**Acceptance Scenarios**:

1. **Given** 動画が開かれており SRT/ASS 字幕ファイルが読み込まれている、**When** 「字幕からブックマーク生成」を実行する、**Then** 各字幕エントリが A点=字幕開始時刻・B点=字幕終了時刻・名前=字幕テキストでブックマークとして登録される
2. **Given** 字幕からブックマークを生成しようとしている、**When** すでに同じ動画にブックマークが存在する、**Then** 既存ブックマークを残したまま字幕分のブックマークを末尾に追記する
3. **Given** 字幕に A点 >= B点 となるエントリが含まれる（0ms 字幕など）、**When** 生成を実行する、**Then** 当該エントリをスキップし有効な字幕は正常に登録、スキップ件数を通知する
4. **Given** 字幕が読み込まれていない状態、**When** 「字幕からブックマーク生成」を実行しようとする、**Then** 「字幕ファイルが読み込まれていません」とエラーメッセージを表示する
5. **Given** 動画が開かれていない状態、**When** 「字幕からブックマーク生成」にアクセスしようとする、**Then** メニュー項目が無効化されている
6. **Given** 一括生成が完了した、**When** Ctrl+Z を押す、**Then** 生成されたブックマークが全件削除される（元に戻す対応）

---

### User Story 2 — すべての練習データをバックアップする (Priority: P2)

PC 移行や定期保存のため「ファイル > データをバックアップ...」からブックマーク・設定・再生位置記憶・最近のファイルをひとつの ZIP ファイルとして保存する。

**Why this priority**: ブックマークは長期の練習成果であり、PC 故障や移行による消失は重大な損失。バックアップ単体で完結した価値を持ち、復元（US3）より先に実装できる。

**Independent Test**: 「データをバックアップ」を実行し、保存先に ZIP が作成され、その中に全データファイルが含まれることを確認できる。

**Acceptance Scenarios**:

1. **Given** アプリが正常動作している、**When** 「ファイル > データをバックアップ...」を実行して保存先を指定する、**Then** `looplayer-backup-YYYYMMDD-HHMMSS.zip` が作成され、ブックマーク・設定・再生位置・最近のファイルの全データが含まれる
2. **Given** バックアップが作成された、**When** ZIP の内容を確認する、**Then** 元のデータが標準的な形式（JSON）で保持されており、looplay! バックアップであることを識別できるマニフェストファイルが含まれる
3. **Given** バックアップ先ディレクトリへの書き込み権限がない、**When** バックアップを実行する、**Then** エラーメッセージを表示し既存データは変更されない

---

### User Story 3 — バックアップからデータを復元する (Priority: P3)

PC 移行後または誤削除後のユーザーが「ファイル > データを復元...」から looplay! バックアップ ZIP を選択すると、確認後にすべての練習データが復元される。

**Why this priority**: バックアップ（US2）が存在することが前提。復元は破壊的操作のためユーザー確認が必要で、US2 より実装・テストが複雑になる。

**Independent Test**: US2 で作成した ZIP を選択して復元を実行し、ブックマーク・設定が元の状態に戻ることを確認できる。

**Acceptance Scenarios**:

1. **Given** looplay! バックアップ ZIP ファイルが存在する、**When** 「ファイル > データを復元...」でその ZIP を選択する、**Then** 「現在のデータはすべて上書きされます。よろしいですか？」という確認ダイアログを表示する
2. **Given** ユーザーが復元を確認した、**When** 復元処理が完了する、**Then** バックアップ時点のブックマーク・設定・再生位置・最近のファイルがすべて復元され、「復元が完了しました。アプリを再起動してください」というメッセージを表示してアプリを終了する
3. **Given** looplay! 形式でない ZIP ファイルを選択した、**When** 復元を実行しようとする、**Then** 「このファイルは looplay! バックアップではありません」とエラーメッセージを表示する
4. **Given** 復元先への書き込み権限がない、**When** 復元を実行する、**Then** エラーメッセージを表示し既存データは変更されない

---

### Edge Cases

- 字幕ファイルのエンコーディングが UTF-8 以外（Shift-JIS 等）の場合は文字化けせずエラーメッセージを表示する
- 字幕テキストが 80 文字を超える場合は先頭 80 文字 + 「...」でブックマーク名を切り詰める
- 字幕区間が重複・入れ子になっている場合は各エントリを独立したブックマークとして登録する
- バックアップ対象データファイルが一つも存在しない場合は空の ZIP を作成せずメッセージを表示する
- バックアップ ZIP が破損している場合は復元開始前にエラーを表示し既存データを保護する
- 字幕生成後のブックマーク件数が非常に多い場合（500件超）でも操作がフリーズしない

## Requirements *(mandatory)*

### Functional Requirements

**F-202: 字幕からブックマーク自動生成**

- **FR-001**: 「字幕からブックマーク生成」は「再生 > 字幕」サブメニュー内（「字幕ファイルを開く...」の下）に配置し、外部字幕ファイルが読み込まれ、かつ動画が開かれている場合にのみ有効化しなければならない
- **FR-002**: 字幕の各エントリを A点=開始時刻・B点=終了時刻・名前=字幕テキストとしてブックマークに一括登録しなければならない
- **FR-003**: ASS 形式の字幕は装飾タグ（`{\an8}` 等）を除去してテキストのみをブックマーク名に使用する
- **FR-004**: A点 >= B点 となる無効なエントリはスキップし、スキップ件数を完了メッセージに含めて通知する
- **FR-005**: 字幕テキストが 80 文字を超える場合は先頭 80 文字 + 「...」に切り詰めてブックマーク名とする
- **FR-006**: 生成されたブックマークは重複チェックなしで既存ブックマークの末尾に追記し、既存ブックマークは変更しない
- **FR-007**: 生成完了後に「N 件のブックマークを生成しました（M 件スキップ）」という成功メッセージを表示する
- **FR-008**: 一括生成操作は Ctrl+Z で元に戻せる（全件削除）

**F-402: データの一括バックアップ・復元**

- **FR-009**: 「ファイル > データをバックアップ...」メニューからバックアップを実行できなければならない
- **FR-010**: バックアップは `looplayer-backup-YYYYMMDD-HHMMSS.zip` という名前の ZIP ファイルとして保存する
- **FR-011**: バックアップ ZIP にはブックマーク・設定・再生位置記憶・最近のファイルの全データと、looplay! バックアップであることを識別するマニフェストファイルを含む
- **FR-012**: 「ファイル > データを復元...」メニューから復元を実行できなければならない
- **FR-013**: 復元前に「現在のデータはすべて上書きされます。よろしいですか？」という確認ダイアログを表示し、ユーザーが拒否した場合は処理を中断する
- **FR-014**: 選択した ZIP ファイルが looplay! バックアップでない場合はエラーメッセージを表示して処理を中断する
- **FR-015**: 復元完了後、「復元が完了しました。アプリを再起動してください」というメッセージを表示してアプリを終了する

### Key Entities

- **SubtitleEntry**: 字幕の1行エントリ。開始時刻（ms）・終了時刻（ms）・テキストを持つ。ブックマーク変換の入力単位
- **BackupArchive**: バックアップ ZIP ファイル。バージョン・作成日時・データファイル群を含む。looplay! バックアップであることを識別するマニフェストを持つ
- **BackupManifest**: ZIP 内に格納されるメタデータ。アプリバージョン・作成日時・含まれるデータファイルの一覧を保持する

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 100 件の字幕を含む SRT ファイルから、5 秒以内にすべてのブックマークが生成される
- **SC-002**: 無効な字幕エントリ（A点 >= B点）がある場合でも、有効なエントリはすべて登録され操作が正常完了する
- **SC-003**: バックアップ操作は 10 秒以内に完了し、ZIP ファイルに全データファイルが含まれる
- **SC-004**: バックアップ → 全データ削除 → 復元 のサイクルで、元のブックマーク・設定が完全に復元される
- **SC-005**: 破損 ZIP や非対応ファイルを指定した場合、既存データが変更されずエラーメッセージが表示される

## Assumptions

- SRT および ASS 形式の字幕を対象とする（外部字幕読み込み F-201 が実装済みであることが前提）
- バックアップ対象は `~/.looplayer/` 以下の全 JSON ファイル（bookmarks.json・settings.json・positions.json・recent_files.json）
- 動画ファイル自体はバックアップ対象外（パス情報のみ保存）
- バックアップ ZIP のマニフェストとして `looplayer-backup.json`（バージョン・作成日時を含む）を ZIP 内に格納する

## Dependencies

- **F-202**: 外部字幕読み込み機能（F-201）が実装済みであること
- **F-402**: 特になし（`~/.looplayer/` の存在のみを前提とする）
