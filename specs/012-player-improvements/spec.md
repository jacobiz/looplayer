# Feature Specification: AB Loop Player Improvements

**Feature Branch**: `012-player-improvements`
**Created**: 2026-03-17
**Status**: Draft
**Input**: ABループ特化プレイヤーとして提案された全機能（A/B点ショートカット、ループ間ポーズ、A/B点微調整、連続再生1周停止、プレイリストUI、フォルダを開くメニュー、練習カウンター、ブックマーク保存時名前入力、タグ付け、クリップ書き出しトランスコード）

## Clarifications

### Session 2026-03-17

- Q: プレイリストのファイル切り替えショートカットが既存の `Ctrl+←/→`（±10秒シーク）と競合する。何のキーを使うか？ → A: `Alt+←` / `Alt+→` を使い、既存キーを維持する
- Q: ポーズ間隔の内部保存単位は何にするか？ → A: ミリ秒（`pause_ms`）で保存。UI スピンボックスは秒表示・内部変換
- Q: フレーム単位微調整ボタンは全行に常時表示するか、アクティブ行のみか？ → A: 全ブックマーク行に常時表示する
- Q: タグフィルタで複数タグを選択した場合の絞り込みロジックは？ → A: OR（いずれかのタグを持つブックマークを表示）

---

## Overview

LoopPlayer の ABループ・ブックマーク体験を総合的に強化する。語学学習・楽器練習に特化した10の機能追加と既存の未実装箇所（フォルダを開くメニュー）の補完により、マウスに頼らないキーボード中心の快適なループ練習環境と、学習進捗の可視化を実現する。

---

## User Scenarios & Testing *(mandatory)*

### User Story 1 - A/B 点キーボードショートカット (Priority: P1)

練習中に動画を見ながら手をキーボードに置いたまま A 点・B 点を設定できる。`I` キーで A 点、`O` キーで B 点を瞬時に打つ。ボタンをマウスでクリックする必要がなくなる。

**Why this priority**: ABループはこのアプリの主機能であり、ループ設定がマウス操作必須というのは根本的な UX 問題。最小工数で最大のユーザー体験改善が見込める。

**Independent Test**: 動画を再生しながら `I` キーを押すと A 点が設定され、`O` キーを押すと B 点が設定され、ループが開始できる。

**Acceptance Scenarios**:

1. **Given** 動画を再生中、**When** `I` キーを押す、**Then** 現在の再生位置が A 点として設定され、「A点セット」ボタンと同じ結果になる
2. **Given** 動画を再生中、**When** `O` キーを押す、**Then** 現在の再生位置が B 点として設定され、「B点セット」ボタンと同じ結果になる
3. **Given** テキスト入力フィールドにフォーカスがある、**When** `I` / `O` キーを押す、**Then** A/B 点が設定されない（入力中はショートカット無効）
4. **Given** ショートカット一覧ダイアログ、**When** 開く、**Then** `I` / `O` のショートカットが一覧に表示される

---

### User Story 2 - A/B 点フレーム単位微調整 (Priority: P1)

再生中にザックリ打った A/B 点を、ブックマーク行の微調整ボタンで 1 フレームずつ精密に合わせられる。音声の始まり・終わりに正確にマークできる。

**Why this priority**: 再生中に正確な点を打つのは難しく、後から精度を上げる手段がないことが現状の大きな制限。語学・楽器練習で正確なループ区間は品質に直結する。

**Independent Test**: ブックマーク行に「A点 −1F」「A点 +1F」「B点 −1F」「B点 +1F」ボタンが表示され、クリックすると 1 フレーム単位で点が移動し再生に反映される。

**Acceptance Scenarios**:

1. **Given** A/B 点が設定されたブックマーク行、**When** 「A点 +1F」ボタンをクリック、**Then** A 点が 1 フレーム分（動画の fps に基づく ms）進む
2. **Given** A/B 点が設定されたブックマーク行、**When** 「B点 −1F」ボタンをクリック、**Then** B 点が 1 フレーム分戻る
3. **Given** A 点を B 点と同じ位置まで進めようとする、**When** 「A点 +1F」を押す、**Then** A >= B になる場合は変更が適用されずエラーを通知する
4. **Given** B 点をファイル末尾を超えようとする、**When** 「B点 +1F」を押す、**Then** 動画の長さを超えない範囲でクランプされる

### User Story 3 - ブックマーク保存時の名前入力 (Priority: P2)

「ブックマーク保存」ボタンを押した瞬間に名前を入力できる。後でダブルクリックしてリネームする手間がなくなる。Enter を押せば自動生成名のまま即保存もできる。

**Why this priority**: 現状は保存後のリネームが必須であり、ブックマーク数が増えると管理が億劫になる。保存フローを 1 ステップに削減する。

**Independent Test**: 「ブックマーク保存」ボタンを押すと名前入力を求める小さなポップアップが表示され、名前を入力して確定するとブックマークが登録される。Enter だけでも保存できる。

**Acceptance Scenarios**:

1. **Given** A/B 点が設定済み、**When** 「ブックマーク保存」をクリック、**Then** 名前入力ポップアップが現れ、デフォルトのブックマーク名（例: 「ブックマーク 3」）が入力済みで表示される
2. **Given** 名前入力ポップアップ表示中、**When** Enter を押す、**Then** デフォルト名でブックマークが保存される
3. **Given** 名前入力ポップアップ表示中、**When** 名前を変更して Enter を押す、**Then** 入力した名前でブックマークが保存される
4. **Given** 名前入力ポップアップ表示中、**When** Escape を押す、**Then** 保存がキャンセルされブックマークは追加されない

---

### User Story 4 - ループ間ポーズ（間隔設定） (Priority: P2)

B 点到達後にすぐ A 点に戻るのではなく、指定した秒数だけ待ってからループを再開できる。「聞いて → 停止 → 真似して発音 → 再生」というシャドーイング練習のリズムを作れる。

**Why this priority**: 語学学習・楽器練習の主要ユースケースで「聞いた後に自分で試す時間」が必要。現状の即ループでは実用が難しい。

**Independent Test**: ブックマーク行に 0〜10 秒の間隔スピンボックスを追加し、B 点到達後に設定した秒数だけ一時停止してから A 点に戻ることを確認できる。

**Acceptance Scenarios**:

1. **Given** ポーズ間隔を 2 秒に設定したブックマークでループ中、**When** B 点に到達する、**Then** 再生が 2 秒間停止し、その後 A 点から再開する
2. **Given** ポーズ間隔が 0 秒（デフォルト）のブックマーク、**When** B 点に到達する、**Then** 従来通り即座に A 点に戻る（既存動作と同一）
3. **Given** ポーズ中に手動でスペースキーを押す、**When** 再生を再開する、**Then** ポーズがキャンセルされ A 点から通常ループを再開する
4. **Given** 連続再生中でポーズ間隔が設定されたブックマーク、**When** B 点に到達する、**Then** 各ブックマークのポーズ間隔が適用される

### User Story 5 - 連続再生の「1周で停止」オプション (Priority: P2)

連続再生を全ブックマーク 1 周完走した時点で自動停止できる。無限ループのみだった連続再生に「1回通し」モードを加える。

**Why this priority**: 練習セッションの終了を手動で止める必要がなくなり、集中力を維持できる。1周後に停止するか無限ループするかは学習スタイルによる基本的な要求。

**Independent Test**: 連続再生ボタンの近くに「1周停止 / 無限ループ」トグルを配置し、1周停止モードで全ブックマークを一巡後に自動停止することを確認できる。

**Acceptance Scenarios**:

1. **Given** 1周停止モードで連続再生中、**When** 最後のブックマークの B 点に到達しリピート回数分ループが完了する、**Then** 再生が停止し連続再生モードが終了する
2. **Given** 無限ループモード（デフォルト）で連続再生中、**When** 最後のブックマークが完了する、**Then** 最初のブックマークに戻り再生が続く（既存動作と同一）
3. **Given** 1周停止モード選択中、**When** 連続再生を開始する、**Then** UI に視覚的にモードが表示される
4. **Given** 連続再生が1周完了して停止した直後、**When** 再び連続再生ボタンを押す、**Then** 最初のブックマークから再スタートする

### User Story 6 - 練習カウンター（再生回数記録） (Priority: P3)

各ブックマークが何回ループ再生されたかを自動カウントし、ブックマーク行に表示する。どのフレーズを何回練習したかが一目でわかる。

**Why this priority**: 学習の積み重ねが可視化されることで継続意欲が高まる。実装コストは小さく、語学学習・楽器練習アプリとしての差別化要素になる。

**Independent Test**: ABループでブックマークを数回再生した後、ブックマーク行に再生回数が表示されることを確認できる。

**Acceptance Scenarios**:

1. **Given** ABループが有効なブックマーク、**When** B 点に到達してループが1回完了する、**Then** そのブックマークの再生回数が1増える
2. **Given** 再生回数が記録されたブックマーク、**When** アプリを再起動してファイルを開く、**Then** 再生回数が保持されている
3. **Given** ブックマーク行、**When** 再生回数が 0 のとき、**Then** 回数は非表示または薄く表示して主要情報を圧迫しない
4. **Given** ブックマーク行の右クリックメニュー、**When** 「再生回数をリセット」を選択する、**Then** そのブックマークの再生回数が 0 にリセットされる

---

### User Story 7 - フォルダを開くメニュー (Priority: P3)

ファイルメニューから「フォルダを開く...」を選択してフォルダを指定できる。現状ドラッグ&ドロップのみで開けるフォルダをメニューからも開けるようにする。

**Why this priority**: i18n にキーが登録済みで実装が抜けている既知の未完成箇所。工数が極めて小さく、ドラッグ&ドロップできない環境（リモートデスクトップ等）でも使える。

**Independent Test**: ファイルメニューに「フォルダを開く...」が表示され、選択するとフォルダ選択ダイアログが開き、フォルダ内の動画ファイルが順再生される。

**Acceptance Scenarios**:

1. **Given** ファイルメニューを開く、**When** 表示する、**Then** 「フォルダを開く...」メニュー項目が「ファイルを開く」の下に存在する
2. **Given** 「フォルダを開く...」をクリック、**When** フォルダを選択して確定する、**Then** フォルダ内の動画ファイルがファイル名順にプレイリストとして読み込まれ再生を開始する
3. **Given** 「フォルダを開く...」をクリック、**When** キャンセルする、**Then** 何も変化しない

---

### User Story 8 - プレイリスト UI パネル (Priority: P3)

フォルダドロップ・フォルダ開きで作成されたプレイリストをパネルで確認し、ファイル一覧から直接選択できる。「次のファイル」「前のファイル」をキーボードで操作できる。

**Why this priority**: プレイリスト機能は実装済みだが UI がなく事実上使えない状態。フォルダ単位で複数の教材を扱う場面に必須。

**Independent Test**: フォルダをドロップするとプレイリストパネルにファイル一覧が表示され、ファイルをクリックすると再生が切り替わる。

**Acceptance Scenarios**:

1. **Given** フォルダをドロップしてプレイリストが読み込まれた、**When** プレイリストパネルを見る、**Then** フォルダ内の動画ファイル一覧が表示され、現在再生中のファイルがハイライトされている
2. **Given** プレイリストパネルが表示されている、**When** 別のファイルをクリックする、**Then** そのファイルの再生が始まる
3. **Given** プレイリストが読み込まれている、**When** `Alt+→` キーを押す、**Then** 次のファイルに切り替わる
4. **Given** プレイリストが読み込まれている、**When** `Alt+←` キーを押す、**Then** 前のファイルに切り替わる
5. **Given** プレイリストが存在しない（単一ファイル再生）、**When** プレイリストパネルエリアを見る、**Then** パネルは非表示またはプレイリストなし状態を表示する

---

### User Story 9 - ブックマークタグ付け (Priority: P4)

ブックマークに「発音」「文法」「難しい」などの自由なタグを付け、タグでフィルタリングして絞り込んで表示・練習できる。

**Why this priority**: 教材が増えると「このカテゴリだけ練習したい」という需要が生まれる。ブックマーク件数が多くなって初めて価値が出るため、他の改善より優先度は低い。

**Independent Test**: ブックマーク行の編集からタグを入力・保存でき、タグフィルタでブックマーク一覧が絞り込まれることを確認できる。

**Acceptance Scenarios**:

1. **Given** ブックマーク行の編集エリア、**When** タグ（例: 「発音」「N2」）を入力して保存する、**Then** ブックマーク行にタグが表示される
2. **Given** 複数のブックマークに異なるタグが付いている、**When** タグフィルタで「発音」を選択する、**Then** 「発音」タグを持つブックマークのみが表示される
3. **Given** タグフィルタで絞り込み中、**When** 連続再生を開始する、**Then** 表示中のブックマーク（フィルタ結果・かつ enabled）のみで連続再生される
4. **Given** タグを持つブックマークを JSON エクスポートする、**When** 別環境にインポートする、**Then** タグ情報も保持されている

---

### User Story 10 - クリップ書き出しのトランスコードオプション (Priority: P4)

現在のコピーモード（高速・フレーム精度が低い）に加えて、再エンコードモード（低速・フレーム精度が高い）を選択できる。H.264 で正確な開始フレームから書き出せる。

**Why this priority**: 語学・練習用クリップの共有でフレーム精度が問題になるケースがある。ただし一般ユーザーの使用頻度は低いため P4。

**Independent Test**: クリップ書き出しダイアログに「高速（コピー）」「正確（再エンコード）」の選択が表示され、それぞれのモードで書き出しが完了する。

**Acceptance Scenarios**:

1. **Given** クリップ書き出しダイアログを開く、**When** 表示する、**Then** 「高速（ストリームコピー）」と「正確（再エンコード・H.264）」の選択肢が表示される
2. **Given** 「高速（ストリームコピー）」を選択、**When** 書き出す、**Then** 現状と同じコピーモードで書き出される（既存動作維持）
3. **Given** 「正確（再エンコード）」を選択、**When** 書き出す、**Then** H.264 + AAC で再エンコードされ指定した開始フレームから正確に書き出される
4. **Given** 選択中のモード、**When** 次回書き出しダイアログを開く、**Then** 前回選択したモードが維持されている

---

### Edge Cases

- A 点と B 点が同一フレームの状態で微調整ボタンを押した場合、A >= B になる変更を拒否する
- ポーズ中にファイルを切り替えた場合、ポーズタイマーはキャンセルされる
- プレイリストのファイルが削除・移動されている場合、そのエントリはスキップし次のファイルへ進む
- タグフィルタで絞り込んだ結果が 0 件の場合、「該当するブックマークがありません」と表示する
- 連続再生1周停止モードで enabled なブックマークが 0 件の場合、開始時にエラーを通知する
- 再エンコード書き出し中にキャンセルした場合、不完全な出力ファイルを削除する
- `I`/`O` ショートカット使用時に動画が読み込まれていない場合は何もしない

## Requirements *(mandatory)*

### Functional Requirements

**US1 - A/B 点ショートカット**
- **FR-001**: `I` キーで A 点を設定できる（再生中・一時停止中ともに有効）
- **FR-002**: `O` キーで B 点を設定できる（再生中・一時停止中ともに有効）
- **FR-003**: テキスト入力フィールドにフォーカスがある間は `I`/`O` のショートカットが無効になる
- **FR-004**: ショートカット一覧ダイアログに `I`/`O` の説明が追加される

**US2 - A/B 点フレーム単位微調整**
- **FR-005**: 全ブックマーク行に A 点の +1F / -1F ボタンを常時表示する
- **FR-006**: 全ブックマーク行に B 点の +1F / -1F ボタンを常時表示する
- **FR-007**: 微調整後の A/B 点はブックマークに即時保存され、次回ループ再生に反映される
- **FR-008**: 微調整によって A >= B になる場合は変更を拒否し、ユーザーに通知する

**US3 - ブックマーク保存時の名前入力**
- **FR-009**: 「ブックマーク保存」実行時に名前入力用のインラインポップアップを表示する
- **FR-010**: ポップアップにはデフォルト名（「ブックマーク N」）が入力済みで表示される
- **FR-011**: Enter キーで名前確定、Escape キーでキャンセルができる

**US4 - ループ間ポーズ**
- **FR-012**: `LoopBookmark` にポーズ間隔フィールド `pause_ms`（ミリ秒、0〜10000、デフォルト 0）を追加する
- **FR-013**: ブックマーク行にポーズ間隔スピンボックスを表示する（UI は秒単位で表示、内部は ms に変換して保存）
- **FR-014**: B 点到達後、設定されたポーズ間隔だけ再生を停止してから A 点に戻る
- **FR-015**: ポーズ中にスペースキーを押すとポーズをスキップして A 点から再開する

**US5 - 連続再生1周停止**
- **FR-016**: 連続再生に「1周停止」と「無限ループ」の 2 モードを設ける
- **FR-017**: デフォルトは従来の「無限ループ」モードとする
- **FR-018**: 1周停止モードでは最後のブックマークが完了した時点で連続再生を終了する
- **FR-019**: 選択中のモードは次回起動時も維持される（設定を永続化する）

**US6 - 練習カウンター**
- **FR-020**: `LoopBookmark` に再生回数（`play_count`）フィールドを追加する
- **FR-021**: ABループ（通常・連続再生）で B 点到達ごとに対象ブックマークの再生回数を 1 増やす
- **FR-022**: ブックマーク行に再生回数を表示する（0 のときは非表示または薄く表示）
- **FR-023**: ブックマーク行の右クリックメニューから再生回数をリセットできる
- **FR-024**: 再生回数はブックマークと同じ JSON ファイルに保存され、アプリ再起動後も保持される

**US7 - フォルダを開くメニュー**
- **FR-025**: ファイルメニューに「フォルダを開く...」メニュー項目を追加する
- **FR-026**: フォルダ選択ダイアログが開き、選択したフォルダの動画ファイルをプレイリストとして読み込む（ドラッグ&ドロップと同一の動作）

**US8 - プレイリスト UI パネル**
- **FR-027**: プレイリスト読み込み時にプレイリストパネルを表示する
- **FR-028**: パネルにはファイル名の一覧と現在再生中ファイルのハイライトを表示する
- **FR-029**: ファイルをクリックするとそのファイルの再生を開始する
- **FR-030**: `Alt+→` で次のファイル、`Alt+←` で前のファイルに切り替えるショートカットを追加する（既存の `Ctrl+←/→` ±10秒シークとの競合を回避）
- **FR-031**: プレイリストが存在しない場合（単一ファイル再生）はパネルを非表示にする

**US9 - ブックマークタグ付け**
- **FR-032**: `LoopBookmark` にタグリスト（`tags`）フィールドを追加する
- **FR-033**: ブックマーク行の編集から複数タグをカンマ区切りで入力・保存できる
- **FR-034**: ブックマークパネルにタグフィルタ（使用中のタグ一覧から複数選択可）を追加する。複数選択時は OR ロジック（いずれかのタグを持つブックマークを表示）
- **FR-035**: タグフィルタで絞り込んだ場合、連続再生もフィルタ結果のみを対象とする
- **FR-036**: タグ情報は JSON エクスポート/インポート時も保持される

**US10 - クリップ書き出しトランスコード**
- **FR-037**: クリップ書き出し時に「高速（ストリームコピー）」と「正確（再エンコード・H.264）」を選択できる
- **FR-038**: デフォルトは「高速（ストリームコピー）」（既存動作を維持）
- **FR-039**: 「正確（再エンコード）」を選択した場合、H.264 + AAC でトランスコードする
- **FR-040**: 選択したモードは次回書き出し時にも維持される

### Key Entities

- **LoopBookmark**（拡張）: 既存フィールドに加えて `pause_ms: int`（ポーズ間隔ミリ秒・デフォルト 0、UI は秒換算で表示）、`play_count: int`（再生回数・デフォルト 0）、`tags: list[str]`（タグリスト・デフォルト []）を追加。既存 JSON に新フィールドがない場合はデフォルト値を使う（後方互換）
- **Playlist**: 動画ファイルパスのリスト＋現在インデックス。フォルダ開き・ドロップで生成。
- **SequentialPlayMode**: 連続再生モード（無限ループ / 1周停止）の設定値。設定ファイルに永続化。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: A/B 点の設定・ブックマーク保存をキーボードのみで完結できる（マウスに触れずにループ設定→保存まで操作できる）
- **SC-002**: ブックマーク保存からループ練習開始まで 3 ステップ以内（点設定 → 名前入力確定 → ループ開始）
- **SC-003**: ループ間ポーズが指定秒数（±200ms 以内）で動作する
- **SC-004**: プレイリスト内のファイル切り替えがキーボード操作で 1 アクションで完了する
- **SC-005**: 練習回数がアプリ再起動後も 100% 保持される
- **SC-006**: 既存の全テストが引き続きパスする（既存機能の回帰なし）
- **SC-007**: 各ユーザーストーリーが独立してテスト可能であり、他の US が未実装でも動作する

## Assumptions

- `I`/`O` ショートカットは VLC プレイヤー等の業界標準に倣う
- ポーズ間隔の上限は 10 秒（練習用途では十分）
- フレーム単位微調整は現在の動画の fps を使用し、fps が取得できない場合は 25fps にフォールバック
- プレイリスト UI はブックマークパネルと同じエリアにタブ切り替えで表示する（レイアウト変更最小化）
- タグの入力 UI はメモ編集と同じ「鉛筆ボタン」を拡張する形で実装する
- 再エンコード時のデフォルト出力品質は一般的な中品質設定（ffmpeg デフォルト相当）
- `LoopBookmark` の新フィールドは後方互換を維持する（既存 JSON に新フィールドがなければデフォルト値を使う）
