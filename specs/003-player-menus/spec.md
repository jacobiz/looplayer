# Feature Specification: プレイヤーメニュー基本機能

**Feature Branch**: `003-player-menus`
**Created**: 2026-03-15
**Status**: Draft
**Input**: 動画プレイヤーとしての体裁をととのえるための各種基本機能のメニュー群を追加したい

## Clarifications

### Session 2026-03-15

- Q: 音量の現在値をどのような UI 要素で表示するか → A: コントロールバーに音量スライダー（%表示ラベル付き）を追加
- Q: フルスクリーン切り替え時にブックマークパネル・コントロール類をどう扱うか → A: 全て非表示（映像のみ表示）

## Overview

現在のアプリはボタン中心の UI で、一般的なデスクトップアプリとしての体裁（メニューバー・キーボードショートカット・音量調整・シーク操作・フルスクリーン）が不足している。本機能追加により、ユーザーがマウスを使わずキーボードだけで主要操作を行えるようになり、また初めて触れるユーザーでも標準的なメニュー構成から機能を探せるようになる。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - ファイルメニューとキーボードショートカット (Priority: P1)

ユーザーはメニューバーの「ファイル」メニューからファイルを開いたり、アプリを終了したりできる。また、よく使う操作（再生・停止・開く）をキーボードショートカットで実行できる。

**Why this priority**: メニューバーは「プレイヤーらしさ」の最も基本的な要素。キーボードショートカットは再生・停止のたびにマウスを動かす手間をなくし、操作効率を大きく改善する。

**Independent Test**: メニューバーを開き「ファイル > 開く」でファイルを選択できること、`Space` キーで再生/一時停止が切り替わること、`Ctrl+Q` でアプリが終了することを確認することで独立してテスト可能。

**Acceptance Scenarios**:

1. **Given** アプリが起動している, **When** メニューバーの「ファイル」を開く, **Then** 「開く」「終了」の項目が表示される
2. **Given** ファイルメニューが開いている, **When** 「開く」を選択する, **Then** ファイル選択ダイアログが表示される
3. **Given** 動画が再生中, **When** `Space` キーを押す, **Then** 再生が一時停止する
4. **Given** 動画が一時停止中, **When** `Space` キーを押す, **Then** 再生が再開する
5. **Given** アプリが起動している, **When** `Ctrl+O` を押す, **Then** ファイル選択ダイアログが表示される
6. **Given** アプリが起動している, **When** `Ctrl+Q` を押す, **Then** アプリが終了する

---

### User Story 2 - 再生コントロールメニューとシーク操作 (Priority: P2)

ユーザーはメニューバーの「再生」メニューから音量調整・ミュート・前後へのシーク・再生速度変更を操作できる。またこれらをキーボードショートカットでも実行できる。

**Why this priority**: 音量調整とシーク操作は動画視聴で毎回使う基本機能。現在はこれらの手段が存在しないため、使い勝手に大きな影響がある。

**Independent Test**: 「再生」メニューから音量を上げ下げできること、`←` / `→` キーで5秒前後にシークできること、再生速度を1.5倍にして動画が速く再生されることを確認する。

**Acceptance Scenarios**:

1. **Given** 動画が再生中, **When** 「再生 > 音量を上げる」を選択する, **Then** 音量が上昇する
2. **Given** 動画が再生中, **When** 「再生 > ミュート」を選択する, **Then** 音声が消え、再選択で復元される
3. **Given** 動画が再生中, **When** `↑` キーを押す, **Then** 音量が上昇する
4. **Given** 動画が再生中, **When** `↓` キーを押す, **Then** 音量が低下する
5. **Given** 動画が再生中, **When** `M` キーを押す, **Then** ミュート状態が切り替わる
6. **Given** 動画が再生中, **When** `→` キーを押す, **Then** 再生位置が5秒前進する
7. **Given** 動画が再生中, **When** `←` キーを押す, **Then** 再生位置が5秒後退する
8. **Given** 動画が再生中, **When** 「再生 > 再生速度 > 1.5倍」を選択する, **Then** 動画が1.5倍速で再生される
9. **Given** 再生速度が変更済み, **When** 「再生 > 再生速度 > 標準（1.0倍）」を選択する, **Then** 通常速度に戻る

---

### User Story 3 - 表示メニューとフルスクリーン (Priority: P3)

ユーザーはメニューバーの「表示」メニューからフルスクリーン表示への切り替えと、ウィンドウを常に最前面に表示する設定を操作できる。

**Why this priority**: フルスクリーン視聴は動画プレイヤーの基本機能。常に最前面表示は BGM や学習用途で需要がある。

**Independent Test**: `F` キーまたはメニューでフルスクリーンに切り替わり、`Esc` キーで元のウィンドウサイズに戻ることを確認する。

**Acceptance Scenarios**:

1. **Given** 通常ウィンドウ表示中, **When** 「表示 > フルスクリーン」を選択する, **Then** 映像が画面全体に広がり、メニューバー・コントロール群・ブックマークパネルが全て非表示になる
2. **Given** フルスクリーン表示中, **When** `Esc` キーを押す, **Then** 元のウィンドウサイズに戻る
3. **Given** 通常ウィンドウ表示中, **When** `F` キーを押す, **Then** フルスクリーンに切り替わる
4. **Given** フルスクリーン表示中, **When** `F` キーを押す, **Then** 通常ウィンドウに戻る
5. **Given** アプリが起動している, **When** 「表示 > 常に最前面に表示」を有効にする, **Then** 他のウィンドウを前面に出してもプレイヤーが最前面を維持する
6. **Given** 常に最前面設定が有効, **When** 「表示 > 常に最前面に表示」を再選択する, **Then** 通常のウィンドウ重ね順に戻る

---

### Edge Cases

- フルスクリーン中にファイルを開こうとした場合、ダイアログが正常に表示されるか
- 動画を開く前にシーク操作キーを押した場合、何も起きずエラーも出ない
- 最大音量時に「音量を上げる」を押しても音量が上限を超えない
- 最小音量（0）時に「音量を下げる」を押しても音量が負にならない
- 再生速度変更後に別のファイルを開いた場合、速度が標準（1.0倍）にリセットされる
- フルスクリーン中にアプリを閉じようとした場合、正常に終了できる
- フルスクリーン中に AB ループが動作中の場合、通常ウィンドウに戻っても AB ループ状態が維持される

## Requirements *(mandatory)*

### Functional Requirements

**ファイルメニュー（FR-001〜FR-003）**

- **FR-001**: アプリはメニューバーに「ファイル」「再生」「表示」の3メニューを持たなければならない
- **FR-002**: 「ファイル > 開く」は既存のファイル選択機能を呼び出さなければならない（`Ctrl+O` ショートカット付き）
- **FR-003**: 「ファイル > 終了」でアプリを終了できなければならない（`Ctrl+Q` ショートカット付き）

**キーボードショートカット（FR-004〜FR-007）**

- **FR-004**: `Space` キーで再生/一時停止をトグルできなければならない
- **FR-005**: `→` キーで再生位置を5秒前進、`←` キーで5秒後退できなければならない
- **FR-006**: `↑` キーで音量を10%上昇、`↓` キーで10%低下できなければならない
- **FR-007**: `M` キーでミュート状態をトグルできなければならない

**再生メニュー（FR-008〜FR-012）**

- **FR-008**: 「再生」メニューに「再生/一時停止」「停止」「音量を上げる」「音量を下げる」「ミュート」「再生速度」が含まれなければならない
- **FR-009**: 音量は0〜100%の範囲で管理され、コントロールバーに音量スライダーと現在値（%表示ラベル）を常時表示しなければならない
- **FR-010**: ミュートは現在の音量を記憶し、解除時に元の音量に戻らなければならない
- **FR-011**: 「再生速度」サブメニューは 0.5 / 0.75 / 1.0 / 1.25 / 1.5 / 2.0 倍の選択肢を持たなければならない
- **FR-012**: 新しいファイルを開いた際に再生速度は標準（1.0倍）にリセットされなければならない

**表示メニュー（FR-013〜FR-016）**

- **FR-013**: 「表示 > フルスクリーン」でウィンドウをフルスクリーンに切り替えられなければならない（`F` キーショートカット付き）。フルスクリーン中はメニューバー・コントロールボタン群・音量スライダー・ブックマークパネルを全て非表示にし、映像のみを表示する
- **FR-014**: フルスクリーン中に `Esc` キーまたはメニュー再選択で通常ウィンドウに戻れなければならない
- **FR-015**: 「表示 > 常に最前面に表示」はトグル動作で、有効時はメニュー項目にチェックマークが付かなければならない
- **FR-016**: フルスクリーン中はメニューバーを非表示とし、マウスカーソルを画面上部に移動したときのみ表示されなければならない

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 主要操作（開く・再生・停止・音量・シーク）の全てにキーボードショートカットが提供され、マウスを一切使わずに動画を視聴開始から終了まで操作できる
- **SC-002**: メニュー項目の選択から操作の反映まで視覚的な遅延がないこと（ユーザーが即時フィードバックを体感できる）
- **SC-003**: フルスクリーンへの切り替えとウィンドウ復帰が 1 回の操作（キー 1 押しまたはメニュー 1 選択）で完了する
- **SC-004**: 音量と再生速度の現在値が常に UI 上で確認できる
- **SC-005**: 動画を開いていない状態でショートカットキーを押してもエラーダイアログが表示されない（無視される）

## Assumptions

- 音量制御は OS レベルではなくアプリ内の再生音量として実装する
- 既存のボタン群（開く・再生・停止など）は引き続き残す（メニューと併存）
- 最近使ったファイルの一覧表示は本フィーチャーのスコープ外とする
- フルスクリーン中のコントロールオーバーレイ（マウスホバーで表示）は本フィーチャーのスコープ外とし、FR-016 のメニューバー表示のみ実装する
