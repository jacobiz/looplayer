# looplay! 不足機能 提案レポート

**作成日**: 2026-03-18
**最終更新**: 2026-03-19
**対象バージョン**: v1.8.1
**分析ベース**: specs/001〜018 の全仕様書

---

## 現状の把握

looplay! は ABループ再生に特化した動画プレイヤーで、以下の機能が実装済みです。

| カテゴリ | 実装済み機能 |
|---------|------------|
| 再生コア | ファイルオープン、再生/停止/シーク、シークバークリック、再生速度変更（0.25x–3.0x・10段階、Shift+[/] で 0.05x 微調整）、コマ送り |
| ABループ | A/B 点設定、ループ再生、キーボードショートカット（I/O）、フレーム単位微調整、シークバードラッグ操作、シークバーズーム表示 |
| ブックマーク | 複数 AB ループの保存・切替・連続再生、チェックボックス選択、メモ、タグ、練習カウンター、ループ間ポーズ、エクスポート/インポート |
| 再生制御 | 音量/ミュート、マルチトラック（音声/字幕）、外部字幕読み込み（SRT/ASS）、再生終了時動作設定、再生位置記憶 |
| UI/UX | フルスクリーン（コントロールオーバーレイ付き）、常に最前面、映像左右反転（ミラー表示）、ドラッグ&ドロップ、最近のファイル、動画情報、ショートカット一覧（ツールチップ充実）、日英 UI 切替、ウィンドウ位置・サイズ記憶 |
| 設定 | 設定画面（再生/表示/アップデートタブ）、初回起動オンボーディング |
| ファイル出力 | ABループ区間のクリップ書き出し（コピー/再エンコード） |
| プレイリスト | フォルダドロップ/メニュー、UIパネル、ファイル切替ショートカット |
| 配布 | Windows インストーラ、自動アップデート |

## カテゴリ 1: コアユースケースの深化

### F-102 ABループ遷移のフェードイン/アウト

- **対象ユーザー**: 語学学習者、音楽練習者
- **問題の本質**: B 点からA 点への切り替えが即座なため音声が唐突に切れる感覚がある。文のつながりやフレーズの余韻を聞きながら練習したい場合にストレスになる
- **提案内容**: ループ遷移時に B 点直前 0.3 秒でフェードアウト、A 点直後 0.1 秒でフェードインをオプション適用する。アプリ全体設定として `settings.json` に保存する
- **実装難易度**: 中
- **優先度**: P3
- **既存機能との関係**: player.py の AB ループ判定処理の拡張

---

### F-103 リピート回数の目標設定と達成表示

- **対象ユーザー**: すべての反復練習ユーザー
- **問題の本質**: 練習カウンターは「何回やったか」を記録するが「何回やるか」の目標がない。練習終了の判断基準がないまま繰り返し続けてしまう
- **提案内容**: ブックマークに `target_count` フィールドを追加し、カウンターが目標に達したら色変え・チェックマーク表示で達成を示す。連続再生時に達成済みブックマークを自動スキップするオプションも設ける
- **実装難易度**: 低
- **優先度**: P2
- **既存機能との関係**: `LoopBookmark` の `play_count` フィールド、`BookmarkRow` UI の拡張

---

### F-104 音声波形表示

- **対象ユーザー**: 楽器練習者、語学学習者
- **問題の本質**: 音の区切り・無音部分・発音開始点が視覚的にわからず、A/B 点の精密設定が感覚頼みになる。シークバー上のブックマーク区間表示だけでは不十分
- **提案内容**: シークバー背景に ffprobe で事前生成した波形データを描画する。現在表示範囲のみ描画することでパフォーマンスを確保し、生成データはキャッシュする
- **実装難易度**: 高
- **優先度**: P3
- **既存機能との関係**: `BookmarkSlider` ウィジェットの描画拡張

---

## カテゴリ 2: 競合比較によるギャップ補完

### F-202 字幕からブックマーク自動生成

- **対象ユーザー**: 映画・ドラマで語学学習をする人
- **問題の本質**: 長い動画で手動 A/B 点設定は膨大な手間。SRT ファイルがあれば「1 字幕 = 1 ブックマーク区間」として自動生成できる。Language Reactor などの語学学習特化ツールにはこの機能がある
- **提案内容**: SRT/ASS 読み込み後「字幕からブックマーク生成」を実行し、各字幕のタイムスタンプを A/B 点・字幕テキストをブックマーク名として一括生成する
- **実装難易度**: 中
- **優先度**: P2
- **既存機能との関係**: `BookmarkStore`、外部字幕読み込み（F-201、実装済み）が前提

---

### F-204 キーボードショートカットのカスタマイズ

- **対象ユーザー**: パワーユーザー、他のプレイヤーからの移行ユーザー
- **問題の本質**: ショートカットはハードコード固定で変更不可。VLC/mpv 経験者は操作体系の違いにストレスを感じる
- **提案内容**: `~/.looplayer/keybindings.json` でショートカットを上書き可能にする。最初は UI 設定画面なし、JSON 手動編集から始める
- **実装難易度**: 中
- **優先度**: P3
- **既存機能との関係**: player.py 内の全 QAction/QShortcut 定義

---

## カテゴリ 3: ユーザージャーニーの欠落

### F-301 練習セッション統計

- **対象ユーザー**: 継続的に練習するすべてのユーザー
- **問題の本質**: 練習カウンターはブックマーク単位の累積回数のみ。「今日どれくらい練習したか」「先週より上達しているか」が時間軸でわからない。「振り返り」ステップが練習サイクルから完全に抜けている
- **提案内容**: 練習ログ（日付・動画ファイル・ブックマーク・再生回数）を `~/.looplayer/practice_log.json` に記録する。日別・ブックマーク別の統計をダイアログで表示する
- **実装難易度**: 中
- **優先度**: P3
- **既存機能との関係**: 練習カウンター（spec 012 US6）の発展形

---

### F-302 習熟度マーキング（未習/練習中/習得済）

- **対象ユーザー**: 語学学習者、楽器練習者
- **問題の本質**: タグは自由テキストで構造化されていない。「まだ難しい」「ほぼマスターした」という段階を管理する仕組みがなく、間隔反復学習（SRS）的なアプローチが欠けている
- **提案内容**: ブックマークに 3 段階ステータス（未習/練習中/習得済）を追加する。連続再生時に「練習中」のみを抽出するフィルタを設ける
- **実装難易度**: 低
- **優先度**: P2
- **既存機能との関係**: `LoopBookmark` へのフィールド追加、タグフィルタと同様のフィルタリング UI

---

### F-303 シャドーイング録音モード

- **対象ユーザー**: 発音練習者、語学学習者
- **問題の本質**: 「手本を見る→自分でやる」サイクルはあるが、「自分の音声を記録して手本と比較する」ステップがない。練習の質的な振り返りができない
- **提案内容**: ループ間ポーズ中に音声録音を自動開始し、次のループ再生前に自分の録音を再生する「シャドーイング録音モード」を追加する。録音はブックマークに紐づけて保存する
- **実装難易度**: 高
- **優先度**: P3
- **既存機能との関係**: ループ間ポーズ（spec 012 US4）との連携

---

## カテゴリ 4: デスクトップアプリとしての完成度

### F-402 データの一括バックアップ・復元

- **対象ユーザー**: 長期利用ユーザー、PC 移行ユーザー
- **問題の本質**: ブックマーク・設定・再生位置・最近のファイルが `~/.looplayer/` に分散保存されているが、一括バックアップ・復元の手段がない。ブックマーク単体エクスポートはあるが、全データの持ち出しができない
- **提案内容**: 「ファイル > データをバックアップ」で `~/.looplayer/` 以下を ZIP アーカイブとして保存する。「ファイル > データを復元」で ZIP からリストアする
- **実装難易度**: 低
- **優先度**: P2
- **既存機能との関係**: `~/.looplayer/` 以下の全データファイル

---

### F-404 macOS / Linux 向け配布パッケージ

- **対象ユーザー**: macOS/Linux ユーザー
- **問題の本質**: Windows インストーラは仕様化・実装済みだが、macOS/Linux 向けの正式な配布手段が未定義。Python + VLC の構成上動作は可能なため、ユーザーが手動で環境構築を強いられている
- **提案内容**: macOS 向け `.dmg`、Linux 向け `.AppImage` または Flatpak を GitHub Actions で生成・配布する。OS 固有バグ（フルスクリーン挙動、ファイルパスなど）のテストを含む
- **実装難易度**: 高
- **優先度**: P3
- **既存機能との関係**: spec 006 Windows インストーラの姉妹機能

---

## カテゴリ 5: アクセシビリティ・ユーザビリティ

### F-504 ダークモード対応

- **対象ユーザー**: 長時間・暗い環境で使用するユーザー
- **問題の本質**: 動画は暗い背景での視聴が多いが、明るい UI との落差で目が疲れる。特に夜間や照明を落とした環境での学習に影響する
- **提案内容**: 設定でダークテーマを適用するオプションを追加する。PyQt6 の `QPalette` またはスタイルシートで実装する。OS のダークモード設定への自動追従モードも用意する
- **実装難易度**: 中
- **優先度**: P3
- **既存機能との関係**: 設定画面（F-401、実装済み）との連携

## 総括

looplay! は ABループのコア体験（ブックマーク管理・連続再生・フレーム微調整・ポーズ間隔）について高い完成度に達しています。v1.6/1.7 で P1・P2 の主要 UX 機能を完了し、v1.8 で再生速度拡張・ミラー表示（F-101/F-203）を実装しました。残る最大のギャップは以下の 2 点です。

### ① 語学学習ワークフローとの統合不足

外部字幕読み込み（F-201）は実装済み。次のステップとして字幕からのブックマーク自動生成（F-202）を実装することで、映画・ドラマを教材にする語学学習者へのバリューが大きく向上します。

### ② 「練習→上達」の可視化不足

カウンターはあるが目標設定（F-103）・習熟度管理（F-302）・統計（F-301）がなく、上達の実感を得る仕組みが弱い。これらを揃えることで looplay! は「動画プレイヤー」から「練習ツール」に昇格します。
