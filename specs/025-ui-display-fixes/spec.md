# Feature Specification: 表示修正 — サイドパネルデフォルト ON・AB点アイコン改善

**Feature Branch**: `025-ui-display-fixes`
**Created**: 2026-03-23
**Status**: Draft
**Input**: User description: "表示修正。・ブックマークサイドパネルはデフォルトでON ・\"A点セット\" \"B点セット\"のアイコンを、それぞれ始点・終点がよりわかるアイコンに変更"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - アプリ起動時にブックマークサイドパネルが最初から表示される (Priority: P1)

初めてアプリを起動したユーザー、または設定をリセットしたユーザーが、ブックマークサイドパネルを手動で開かなくても最初から表示された状態でアプリを使い始められる。

**Why this priority**: ブックマークパネルはアプリの主要機能（ループ再生・区間管理）への入口であり、デフォルトで非表示だと存在に気づかないユーザーがいる。初期表示をONにすることで発見性が上がる。

**Independent Test**: 設定ファイルを持たない（初回起動相当の）状態でアプリを起動し、ブックマークサイドパネルが表示されていることを確認できる。

**Acceptance Scenarios**:

1. **Given** アプリの設定ファイルが存在しない（初回起動）状態、**When** アプリを起動する、**Then** ブックマークサイドパネルが自動的に表示された状態になっている
2. **Given** ユーザーが手動でサイドパネルを非表示にした状態、**When** アプリを再起動する、**Then** ユーザーの設定（非表示）が保持されている（既存設定の上書きはしない）
3. **Given** アプリの設定ファイルに `bookmark_panel_visible` の記録がある状態、**When** アプリを起動する、**Then** 記録された設定値（ON/OFF）が尊重される

---

### User Story 2 - A点セット・B点セットのアイコンで始点・終点が直感的に判別できる (Priority: P2)

ユーザーが「A点セット」「B点セット」ボタンを見たとき、アイコンだけでどちらが始点（ループ開始）でどちらが終点（ループ終了）かを直感的に識別できる。

**Why this priority**: 現在のアイコン（巻き戻し・早送り）は「A点が始点、B点が終点」という概念を直接表現しておらず、ユーザーが混乱する可能性がある。より意味的に適切なアイコンに変更することで操作ミスを減らせる。

**Independent Test**: ボタンのアイコンを見て、事前説明なしに「左が始点用、右が終点用」と判断できるかを確認できる。

**Acceptance Scenarios**:

1. **Given** アプリが起動している状態、**When** 「A点セット」ボタンを確認する、**Then** ボタンのアイコンが「区間の始点・開始」を連想させるデザインになっている
2. **Given** アプリが起動している状態、**When** 「B点セット」ボタンを確認する、**Then** ボタンのアイコンが「区間の終点・終了」を連想させるデザインになっている
3. **Given** A点・B点ボタンが並んで表示されている状態、**When** 二つのボタンを比較する、**Then** A点（左）が「始まり」、B点（右）が「終わり」を視覚的に表現しており、対として理解しやすい

---

### Edge Cases

- ユーザーが過去に `bookmark_panel_visible = false` を明示的に保存していた場合、デフォルト変更の影響を受けないか？
- 新しいアイコンが OS のテーマ（ダーク/ライト）で視認性を維持できるか？

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: アプリの設定ファイルに `bookmark_panel_visible` の値が存在しない場合、ブックマークサイドパネルはデフォルトで表示状態（ON）でなければならない
- **FR-002**: ユーザーが手動でサイドパネルの表示/非表示を変更した場合、その設定は次回起動時に保持されなければならない（既存動作を維持）
- **FR-003**: 「A点セット」ボタンのアイコンは、現在の「巻き戻し」アイコンから「始点・開始」を表すアイコンに変更されなければならない
- **FR-004**: 「B点セット」ボタンのアイコンは、現在の「早送り」アイコンから「終点・終了」を表すアイコンに変更されなければならない
- **FR-005**: 変更後のA点・B点アイコンは、対として並べたときに「始まりと終わり」の関係が視覚的に伝わるペアデザインでなければならない

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 初回起動時（設定ファイルなし）にブックマークサイドパネルが表示された状態でアプリが起動する
- **SC-002**: 既存の `bookmark_panel_visible` 設定を持つユーザーの設定値が起動後も変わらない（リグレッションなし）
- **SC-003**: A点・B点ボタンのアイコンが変更され、以前の巻き戻し・早送りアイコンが表示されなくなる
- **SC-004**: 変更後のアイコンが OS 標準のアイコンセットから選ばれており、テーマ変更時も適切に表示される

## Assumptions

- `bookmark_panel_visible` のデフォルト値を `False` から `True` に変更するだけでよい（設定ファイル形式の変更は不要）。
- 既存の設定ファイルに `bookmark_panel_visible` キーが存在する場合はその値を優先するため、既存ユーザーへの影響はない。
- A点・B点の新アイコンは OS 標準アイコン（既存の `_apply_btn_icon` の仕組みで適用）から選択する。具体的なアイコン選定は実装フェーズで判断する。
