# Feature Specification: ビデオプレイヤーコア機能

**Feature Branch**: `001-video-player-core`
**Created**: 2026-03-15
**Status**: Draft
**Input**: User description: ビデオプレイヤーのコア機能（ファイル開く・再生制御・シーク・ABループ）

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 動画ファイルの再生 (Priority: P1)

ユーザーがローカルの動画ファイルを選択し、再生・一時停止・停止を操作する。
再生中はシークバーで任意の位置にジャンプでき、現在時刻と総時間を常に確認できる。

**Why this priority**: 動画を再生できなければ他の機能に意味がない。最小限の価値を提供するMVP。

**Independent Test**: ファイルダイアログで動画を開き、再生・一時停止・停止・シークが
正常に動作することを確認するだけで独立して検証できる。

**Acceptance Scenarios**:

1. **Given** アプリが起動している, **When** 「開く」ボタンを押してmp4ファイルを選択する,
   **Then** 動画が自動的に再生を開始し、ウィンドウタイトルにファイル名が表示される
2. **Given** 動画が再生中である, **When** 「再生」ボタン（一時停止と表示）を押す,
   **Then** 動画が一時停止し、ボタンラベルが「再生」に戻る
3. **Given** 動画が一時停止中である, **When** 「再生」ボタンを押す,
   **Then** 動画が再開し、ボタンラベルが「一時停止」になる
4. **Given** 動画が再生中である, **When** 「停止」ボタンを押す,
   **Then** 再生が停止しシークバーが先頭に戻り、時刻表示が「00:00 / 00:00」になる
5. **Given** 動画が再生中である, **When** シークバーをドラッグする,
   **Then** 動画が対応する位置にジャンプし、時刻表示が更新される
6. **Given** 動画が再生中である, **When** 何も操作しない,
   **Then** シークバーが再生進行に合わせて自動更新され、現在時刻/総時間が表示される

---

### User Story 2 - ABループ再生 (Priority: P2)

ユーザーが動画の特定区間（A点〜B点）を繰り返し視聴する。
語学学習・動作分析など、同一区間を何度も確認したいユースケースに対応する。

**Why this priority**: 差別化機能。再生基盤（US1）が前提となるため P2。

**Independent Test**: 動画を開き A点・B点をセットしてABループをONにし、
B点到達時にA点へ自動的に戻ることを確認することで独立して検証できる。

**Acceptance Scenarios**:

1. **Given** 動画が再生中である, **When** 「A点セット」ボタンを押す,
   **Then** 現在位置がA点として記録され、UIに時刻が表示される（例: A: 00:15）
2. **Given** 動画が再生中である, **When** 「B点セット」ボタンを押す,
   **Then** 現在位置がB点として記録され、UIに時刻が表示される（例: B: 00:30）
3. **Given** A点とB点がセットされている, **When** 「ABループ: OFF」ボタンを押す,
   **Then** ボタンが「ABループ: ON」に変わりループが有効になる
4. **Given** ABループがONで動画が再生中である, **When** 再生位置がB点に達する,
   **Then** 自動的にA点に戻り再生が継続する
5. **Given** ABループがONである, **When** 「ABリセット」ボタンを押す,
   **Then** A点・B点がクリアされ、ループがOFFになり、UIが「A: --  B: --」に戻る

---

### Edge Cases

- A点のみセットされB点がセットされていない状態でABループをONにした場合、ループは機能しない
- B点がA点より前の時刻にセットされた場合の動作（不定動作のため制限なし、将来対応）
- 動画を開いていない状態でA点・B点セットボタンを押した場合、何も起こらない
- 動画が終端に達した場合（ABループのON/OFFに関わらず）、再生が自然に停止する。ABループのON/OFF状態は維持される
- 対応していない形式のファイルを選択した場合、ダイアログで選択可能なファイルが制限される
- ファイルが開けない場合（コーデック非対応・ファイル破損・権限エラー等）、エラーダイアログを
  表示し、直前の再生状態を維持する（プレイヤーはリセットしない）

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムはユーザーがファイルダイアログから動画ファイルを選択して開けなければならない
- **FR-002**: システムは mp4, avi, mkv, mov, wmv, flv, webm, m4v 形式の動画を再生できなければならない
- **FR-003**: ユーザーはトグルボタンで再生と一時停止を切り替えられなければならない
- **FR-004**: ユーザーは停止ボタンで再生を停止し先頭に戻せなければならない
- **FR-005**: システムは再生中にシークバーの位置を自動更新しなければならない（更新頻度は約200ms）
- **FR-006**: ユーザーはシークバーをクリックまたはドラッグして任意の再生位置に移動できなければならない
- **FR-007**: システムは現在の再生時刻と総時間を「MM:SS / MM:SS」形式で表示しなければならない
  （1時間以上の場合は「HH:MM:SS / HH:MM:SS」形式）
- **FR-008**: ユーザーは再生中の任意の時点でA点をセットできなければならない
- **FR-009**: ユーザーは再生中の任意の時点でB点をセットできなければならない
- **FR-010**: システムはA点・B点の時刻をUIに表示しなければならない
- **FR-011**: ユーザーはABループのON/OFFをトグルボタンで切り替えられなければならない
- **FR-012**: ABループがONでA点・B点が両方セットされている場合、再生位置がB点に達したとき
  システムは自動的にA点に戻らなければならない
- **FR-013**: ユーザーはABリセットボタンでA点・B点とループ状態を一括クリアできなければならない
- **FR-014**: 新しいファイルを開いたとき、システムはAB点とループ状態を自動的にリセットしなければならない
- **FR-015**: ファイルが開けない場合（コーデック非対応・破損・権限エラー等）、システムはエラーダイアログを
  表示し、直前の再生状態を維持しなければならない
- **FR-016**: ウィンドウリサイズ時、動画はアスペクト比を維持してビデオ表示領域に収まるよう表示されなければ
  ならない（レターボックス/ピラーボックス方式）

### Key Entities

- **再生セッション**: 現在開いているファイルの状態（パス・再生位置・再生/停止状態）
- **AB区間**: A点時刻（ミリ秒）・B点時刻（ミリ秒）・ループ有効フラグの組み合わせ

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーはファイルを選択してから3秒以内に動画の再生を開始できる
- **SC-002**: シークバー操作から再生位置の更新まで体感的に即座（1秒未満）に反映される
- **SC-003**: ABループ機能を使って特定区間を途切れなく繰り返し視聴できる
  （B点到達からA点への戻りが視覚的にスムーズ）
- **SC-004**: 初めて使うユーザーがマニュアルなしでファイル開く・再生・ABループを
  設定できる（直感的なUI）

## Clarifications

### Session 2026-03-15

- Q: ファイルが開けない場合（コーデック非対応・破損・権限エラー）のエラーハンドリングは？ → A: エラーダイアログを表示し、直前の再生状態を維持する
- Q: 動画が終端に達したときの動作（ABループON中含む）は？ → A: 終端で再生停止、ABループのON/OFF状態はそのまま維持
- Q: ウィンドウリサイズ時のビデオ表示方法は？ → A: アスペクト比を維持してビデオ表示領域に収める（レターボックス/ピラーボックス）

## Assumptions

- ローカルファイルシステムからのファイル読み込みのみ対応（URLストリーミングは対象外）
- ABループのB点はA点より後の時刻であることをユーザーが保証する（バリデーションなし）
- 同時に開けるファイルは1つ（マルチウィンドウは対象外）
- 再生速度変更・音量調整・字幕は本フィーチャーの対象外
