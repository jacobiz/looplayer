# Feature Specification: ウィンドウサイズを動画サイズに合わせる機能のバグ修正

**Feature Branch**: `015-fix-window-resize`
**Created**: 2026-03-18
**Status**: Draft
**Input**: User description: "上記1-4のバグ修正"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 動画に合わせてリサイズしたとき映像が正しいアスペクト比で表示される (Priority: P1)

ユーザーがメニューから「ウィンドウサイズを動画サイズに合わせる」を選択したとき、動画エリアが動画本来の解像度通りのアスペクト比で表示される。現在はUIコントロール（シークバー・ボタン類）の高さ分だけウィンドウが小さくリサイズされるため、動画が縦に圧縮されて表示される不具合がある。

**Why this priority**: 機能の本来の目的（動画を正しいサイズ・比率で表示する）が達成できていない最重要バグ。ユーザーが最も気づきやすく影響が大きい。

**Independent Test**: 横長動画（16:9）を開き、「ウィンドウサイズを動画サイズに合わせる」を選択して映像エリアのアスペクト比が崩れていないことを目視および数値で確認できる。

**Acceptance Scenarios**:

1. **Given** 1920×1080 の動画が開かれている状態で、**When** 「ウィンドウサイズを動画サイズに合わせる」を選択する、**Then** 動画エリアが 1920×1080 の比率（または画面サイズにクランプされた同比率）で表示され、映像が縦に圧縮されない
2. **Given** 任意の動画が開かれている状態で、**When** リサイズを実行する、**Then** UIコントロール部分の高さがウィンドウ目標サイズに正しく加算されている
3. **Given** 画面より大きい解像度の動画が開かれている状態で、**When** リサイズを実行する、**Then** 利用可能な画面領域内に収まるようにクランプされる

---

### User Story 2 - 動画が読み込めない状態でもアプリが安定して動作する (Priority: P2)

動画ファイルが壊れている・読み込めないなどの異常時に、サイズポーリングタイマーが永続的に動き続けることなく自動で停止する。

**Why this priority**: 通常操作では発生しにくいが、タイマーが無限に動き続けるとリソースリークになる可能性がある。

**Independent Test**: 動画サイズが取得できない状況を再現し、一定時間後にポーリングが自動停止することを確認できる。

**Acceptance Scenarios**:

1. **Given** 動画サイズが取得できない状態でポーリングが開始されている場合、**When** 5 秒が経過する、**Then** ポーリングタイマーが自動的に停止する
2. **Given** 正常な動画を開いた場合、**When** サイズが取得できた時点で、**Then** ポーリングタイマーが即座に停止する

---

### User Story 3 - デッドコードの除去によりコードが整理される (Priority: P3)

未使用の `_user_resized` フラグと VLC イベントに接続されていない `_on_vlc_video_changed` メソッドを削除し、将来の誤解や誤った修正を防ぐ。

**Why this priority**: 機能的な影響はないが、誤解を招くコードを除去することで保守性が向上する。

**Independent Test**: 削除後も「ウィンドウサイズを動画サイズに合わせる」機能および関連するすべての既存テストが正常に動作することを確認できる。

**Acceptance Scenarios**:

1. **Given** デッドコード削除後の状態で、**When** 全テストスイートを実行する、**Then** 全テストが通過する
2. **Given** デッドコード削除後の状態で、**When** コードを参照する、**Then** `_user_resized` および `_on_vlc_video_changed` が存在しない

---

### Edge Cases

- 動画が開かれていない状態で「ウィンドウサイズに合わせる」を選択した場合、処理が安全にスキップされクラッシュしない
- フルスクリーン中に「ウィンドウサイズに合わせる」を選択した場合、何も起きない（既存動作を維持）
- ウィンドウが最小化されている状態でリサイズが呼ばれた場合、安全に処理される
- UIコントロールが非表示（フルスクリーン中）から表示に切り替わった直後にリサイズ計算が行われる場合、正しいUIオフセットが使用される

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: ウィンドウリサイズ計算は、動画解像度の高さにUIコントロール部分の高さを加算した値を目標ウィンドウ高さとしなければならない
- **FR-002**: サイズポーリングタイマーは、動画サイズが取得できない場合でも最大 5 秒以内に自動停止しなければならない
- **FR-003**: 動画サイズが正常に取得できた時点でポーリングタイマーは即座に停止しなければならない（既存動作を維持）
- **FR-004**: 未使用フラグ `_user_resized` をコードから削除しなければならない
- **FR-005**: VLC イベントに接続されていない `_on_vlc_video_changed` メソッドをコードから削除しなければならない
- **FR-006**: フルスクリーン中はリサイズ処理をスキップしなければならない（既存動作を維持）
- **FR-007**: ウィンドウサイズは利用可能な画面領域を超えないようにクランプしなければならない（既存動作を維持）
- **FR-008**: 動画が開かれていない状態でリサイズが要求された場合、安全にスキップしなければならない

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 「ウィンドウサイズを動画サイズに合わせる」実行後、動画エリアの縦横比が動画ファイルの解像度と一致する（視覚的に歪みがない）
- **SC-002**: 動画が読み込めない状態でも 5 秒以内にポーリングが停止し、それ以降の余分な処理が発生しない
- **SC-003**: リファクタリング（デッドコード削除）後に既存テストスイートが 100% 通過する
- **SC-004**: 修正後に「ウィンドウサイズを動画サイズに合わせる」に関する映像比率の乱れが解消される

## Assumptions

- UIコントロール部分の高さはウィンドウの現在の高さと動画フレームの高さの差分として動的に取得できる
- ポーリングタイムアウトは 5 秒（100ポーリング × 50ms間隔）を標準とする
- `_on_vlc_video_changed` の VLC MediaPlayerVideoChanged イベントへの再接続は今回スコープ外とし、削除のみを行う
- ウィンドウ最小サイズ（幅 800px・高さ 600px）のクランプは既存動作を維持する
