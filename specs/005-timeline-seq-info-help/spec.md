**Status**: Draft

## User Scenarios & Testing *(mandatory)*

### User Story 1 - タイムライン上のABループ区間表示 (Priority: P1)

語学学習者がシークバーを見ただけで、どこにどのブックマーク区間があるかを視覚的に把握できる。現在はシークバーが単色のスライダーのみで、ブックマーク区間の位置が分からない。各ブックマーク区間をシークバー上に色付きバーで表示し、クリックするとその区間（A点）に即ジャンプできる。

**Why this priority**: 複数ABループ連続再生に特化したアプリの核心機能。ブックマークが何件あっても、シークバー一目で全区間の配置が分かることは、練習箇所の把握・選択の基本となる。

**Independent Test**: 動画を開いてブックマークを2件以上登録した状態でシークバーを確認することで、区間表示が機能しているか独立してテストできる。

**Acceptance Scenarios**:

1. **Given** 動画を開きブックマークが3件登録されている状態で、**When** ユーザーがシークバーを視認する、**Then** 各ブックマーク区間がシークバー上に色付きバー（区間の長さに比例した幅）として表示される
2. **Given** シークバー上にブックマーク区間バーが表示されている状態で、**When** ユーザーがそのバーをクリックする、**Then** 再生位置がそのブックマークのA点に移動し、そのブックマークのABループが有効になる（複数区間が重複している箇所では最前面＝最後に登録されたブックマークが選択される）
3. **Given** ブックマークが0件の状態で、**When** ユーザーがシークバーを視認する、**Then** 区間バーは表示されない（通常のシークバーのみ）
4. **Given** 複数のブックマーク区間が重なっている場合、**When** ユーザーがシークバーを視認する、**Then** 重複する区間は同一高さで半透明重ね描きされ、色の違いで識別できる
5. **Given** 連続再生中の状態で、**When** ユーザーがシークバーを視認する、**Then** 現在再生中のブックマーク区間が強調表示される

### User Story 2 - 連続再生対象のチェックボックス選択 (Priority: P1)

ユーザーが複数ブックマークのうち特定の区間だけを選んで連続再生できる。現状は全ブックマークを強制的に全件巡回するのみ。各ブックマーク行にチェックボックスを追加し、チェックされた区間だけを連続再生の対象にする。

**Why this priority**: 「今日は苦手な箇所だけ練習したい」「第2章だけ復習したい」という実際の学習シーンに直結する。全件巡回固定では細かい練習計画を立てられない。

**Independent Test**: ブックマークを3件登録し、そのうち2件にチェックを入れて連続再生を開始することで独立してテストできる。チェックした2件だけが再生されれば成功。

**Acceptance Scenarios**:

1. **Given** ブックマークが複数件ある状態で、**When** ユーザーがブックマーク一覧を確認する、**Then** 各ブックマーク行にチェックボックスが表示されており、デフォルトは全チェック済み（全件選択）
2. **Given** 3件のうち2件にチェックを入れた状態で、**When** 連続再生ボタンを押す、**Then** チェックされた2件のみが順番に連続再生される（未チェックの1件はスキップされる）
3. **Given** チェックが1件もない状態で、**When** ユーザーがブックマーク一覧を確認する、**Then** 連続再生ボタンはグレーアウトされており押せない
4. **Given** 連続再生中に、**When** ユーザーが別のブックマークのチェックを変更する、**Then** 現在の区間の再生完了後から変更後の選択セットに切り替わる
5. **Given** チェック状態が変更された後に、**When** アプリを再起動する、**Then** チェック状態は保持されている（永続化される）

### User Story 3 - 動画情報の表示 (Priority: P2)

ユーザーが現在再生中の動画のメタ情報（解像度・フレームレート・コーデック・ファイルサイズ・動画長）を確認できる。長時間動画や高解像度素材を扱う際に、動画の基本情報をアプリ内で把握したい。

**Why this priority**: 直接的な学習機能ではないが、「この動画はなぜ重い？」「解像度はいくつ？」という疑問に答える補助機能として有用。P1の機能群とは独立して実装・テストできる。

**Independent Test**: 動画を開いた状態で「動画情報」メニューを選択することで独立してテストできる。情報ダイアログが表示されれば成功。

**Acceptance Scenarios**:

1. **Given** 動画を開いている状態で、**When** ユーザーが「動画情報」を選択する、**Then** ファイル名・ファイルサイズ・動画の長さ・解像度（幅×高さ）・フレームレート・映像コーデック・音声コーデックをダイアログで表示する
2. **Given** 動画を開いていない状態で、**When** ユーザーが「動画情報」メニューを確認する、**Then** 「動画情報」メニュー項目はグレーアウト（無効化）されている
3. **Given** 動画情報ダイアログが表示されている状態で、**When** ユーザーがダイアログのテキストを選択する、**Then** テキストはコピー可能である

---

### User Story 4 - キーボードショートカット一覧画面 (Priority: P2)

ユーザーが利用可能なキーボードショートカットを一覧で確認できる。現状はショートカットをメニューのアクセラレーターキー表示から推測するしかなく、メニューにない操作のショートカット（←/→5秒スキップなど）は発見困難。

**Why this priority**: ショートカット一覧は学習効率を上げる補助機能。新規ユーザーや慣れていないユーザーが全機能を活用するための助けになる。P1機能群と完全に独立している。

**Independent Test**: `?` キーを押す（またはメニューから「ショートカット一覧」を選択）ことで独立してテストできる。全ショートカットを一覧したダイアログが表示されれば成功。

**Acceptance Scenarios**:

1. **Given** アプリ起動中の任意の状態で、**When** ユーザーが `?` キーを押す、**Then** 全ショートカットキーの一覧をカテゴリ別に表示するダイアログが開く
2. **Given** ショートカット一覧ダイアログが表示されている状態で、**When** ユーザーがダイアログの内容を確認する、**Then** 再生操作・音量操作・ABループ操作・ブックマーク操作・表示操作・ファイル操作の各カテゴリでショートカットがまとめられている
3. **Given** ショートカット一覧ダイアログが表示されている状態で、**When** ユーザーが `Escape` または閉じるボタンを押す、**Then** ダイアログが閉じられ、動画再生は継続される
4. **Given** アプリ起動中の任意の状態で、**When** ユーザーが「ヘルプ」メニューから「ショートカット一覧」を選択する、**Then** 同じショートカット一覧ダイアログが開く

---

### Edge Cases

- ブックマーク区間が動画の長さより後に設定されていた場合（動画を差し替えた場合など）、タイムラインバーはどう表示されるか？ → 動画長を超える区間はクリップして非表示にする（シンプルさ優先）
- 連続再生中に、再生対象の全チェックが外れた場合 → 現在の区間が終わったタイミングで連続再生を自動停止する
- 動画情報の取得がタイムアウトまたは失敗した場合 → 取得できた項目だけ表示し、取得できなかった項目は「不明」と表示
- ブックマーク区間が非常に短い（1秒未満）場合、タイムラインバーの幅が極小になる → 最小幅を設けてクリック可能な大きさを確保する
- フルスクリーン中にショートカット一覧ダイアログを開いた場合 → ダイアログは最前面に表示され、閉じてもフルスクリーン状態が維持される

## Requirements *(mandatory)*

### Functional Requirements

**タイムライン上のABループ区間表示**

- **FR-001**: シークバー上に、現在の動画に紐づく各ブックマークのA点〜B点区間をカラーバーとして重ねて表示しなければならない。区間が重複する場合は同一高さで半透明重ね描きし、各ブックマークに割り当てた固有色で識別できるようにする
- **FR-002**: タイムラインバーのクリックにより、対応するブックマークのA点に再生位置をジャンプし、そのブックマークのABループを有効にしなければならない
- **FR-003**: 連続再生中、現在再生中のブックマーク区間バーを他の区間と視覚的に区別して強調表示しなければならない
- **FR-004**: ブックマークの追加・削除・並び替えに応じて、タイムラインバーはリアルタイムで更新されなければならない
- **FR-005**: 区間バーが極小サイズになる場合でも、最小クリック可能幅を確保しなければならない

**連続再生対象のチェックボックス選択**

- **FR-006**: 各ブックマーク行にチェックボックスを設け、チェック状態のブックマークのみを連続再生対象とする
- **FR-007**: チェックボックスのデフォルト状態は「全チェック済み」とする（新規ブックマーク追加時も含む）
- **FR-008**: チェックが1件もない状態では連続再生ボタンをグレーアウト（無効化）しなければならない
- **FR-009**: チェック状態は動画ファイルごとにブックマークデータとともに永続化されなければならない
- **FR-010**: 連続再生中にチェック状態が変更された場合、現在区間終了後から変更後の選択セットを反映する

**動画情報の表示**

- **FR-011**: 動画を開いている状態で「動画情報」を選択すると、ファイル名・ファイルサイズ・動画の長さ・解像度（幅×高さ）・フレームレート・映像コーデック・音声コーデックを表示するダイアログを開かなければならない
- **FR-012**: 動画を開いていない場合は「動画情報」メニュー項目を無効化しなければならない
- **FR-013**: 動画情報ダイアログ内のテキストはコピー可能でなければならない

**キーボードショートカット一覧**

- **FR-014**: `?` キーの押下で、全ショートカットをカテゴリ別にまとめた一覧ダイアログを表示しなければならない
- **FR-015**: ショートカット一覧はヘルプメニューからも呼び出せなければならない
- **FR-016**: 一覧は「再生操作」「音量操作」「ABループ操作」「ブックマーク操作」「表示操作」「ファイル操作」のカテゴリに整理されなければならない
- **FR-017**: ショートカット一覧ダイアログを閉じても再生中の動画は継続されなければならない

### Key Entities

- **ブックマーク（LoopBookmark）**: 既存エンティティ。連続再生対象かどうかを示す `enabled` 属性を追加する
- **タイムラインバー**: シークバー上にオーバーレイ描画される区間表示要素。ブックマークIDと対応し、A点・B点・動画長さから相対位置を計算して表示する
- **動画メタ情報**: 動画ファイルから取得する読み取り専用情報（解像度、フレームレート、コーデック、ファイルサイズ、動画長）

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ブックマーク区間を持つ動画を開いたとき、シークバーに全区間バーが表示されるまで1秒以内に完了する
- **SC-002**: タイムラインバーのクリックからABループ開始まで、視覚的な応答が0.5秒以内である
- **SC-003**: チェックボックスで2件を選択して連続再生した場合、未選択のブックマークが一切再生されない（誤再生率0%）
- **SC-004**: 動画情報ダイアログに表示される7項目がすべて正しく表示される（ファイル名・サイズ・長さ・解像度・FPS・映像コーデック・音声コーデック）
- **SC-005**: ショートカット一覧ダイアログに、既存の全ショートカットが漏れなく掲載されている
- **SC-006**: チェック状態がアプリ再起動後も保持されている（永続化の確認）

## Clarifications

### Session 2026-03-16

- Q: タイムラインバーの重複区間の描画方式は？ → A: 同一高さで半透明重ね描き（スタック表示なし、色の違いで識別）
- Q: 重複バーをクリックしたとき、どのブックマークが選択されるか？ → A: 最前面（最後に登録された）ブックマークを選択
- Q: チェックが0件のときの連続再生ボタンの動作は？ → A: ボタンをグレーアウト（無効化）

## Assumptions

- タイムラインバーの色はブックマーク間で自動的に異なる色が割り当てられる（ユーザーが色を選択する機能は本スコープ外）
- 動画情報の取得は既存のメディアプレイヤーが提供する範囲に限定する（外部ツールによる詳細解析は対象外）
- チェックボックスの状態は既存の `bookmarks.json` に追加フィールドとして保存する（新規ファイルは作成しない）
- ショートカット一覧は静的なテキスト表示のみ（カスタマイズ機能は対象外）
- `?` キーのショートカットはフルスクリーン中を含む全状態で有効とする
