# Feature Specification: P1優先度機能の実装

**Feature Branch**: `016-p1-features`
**Created**: 2026-03-18
**Status**: Draft
**Input**: User description: "@specs/feature.md から優先度P1の機能を実装して"

## Clarifications

### Session 2026-03-18

- Q: 別の動画ファイルを開いたとき、読み込み済みの外部字幕はどうなるか？ → A: リセット（次の動画では字幕なし状態になる）
- Q: フルスクリーン状態でアプリを終了した場合、次回起動時に何を復元するか？ → A: フルスクリーン移行前に記憶した最後のウィンドウ位置・サイズを復元する
- Q: 非 UTF-8 エンコーディングの字幕ファイル（Shift-JIS など）を選択した場合どう扱うか？ → A: UTF-8 のみ対応。「UTF-8 形式の字幕ファイルを使用してください」とエラーを表示する

---

## Overview

本仕様は `specs/feature.md` で優先度 P1 と定義された以下の 3 機能を一括実装するものです。

| ID | 機能名 | 難易度 |
|----|--------|--------|
| F-201 | 外部字幕ファイルの読み込み（SRT/ASS） | 低 |
| F-403 | ウィンドウ位置・サイズの記憶 | 低 |
| F-502 | ツールチップの充実 | 低 |

---

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 外部字幕ファイルの読み込み（F-201）(Priority: P1)

語学学習者が映画・ドラマ動画に対して別途用意した SRT/ASS 字幕ファイルを読み込み、動画上に字幕を表示しながら AB ループ練習ができる。

**Why this priority**: 語学学習者はアプリの最大ユーザー層であり、外部字幕は標準的な学習ワークフローの必須要素。実装難易度が低く、効果が最大のギャップを埋める。

**Independent Test**: メニューから「字幕ファイルを開く」を選択し、SRT ファイルを指定することで字幕が表示される。この機能単独で語学学習者へ価値を届けられる。

**Acceptance Scenarios**:

1. **Given** 動画が再生中で字幕が表示されていない状態で、**When** メニューから SRT ファイルを選択すると、**Then** 動画上に字幕が表示される。
2. **Given** 動画が再生中で字幕が表示されていない状態で、**When** メニューから ASS ファイルを選択すると、**Then** スタイル付き字幕が動画上に表示される。
3. **Given** 字幕ファイルが読み込まれた状態で、**When** 既存の字幕メニューで字幕トラックを切り替えると、**Then** 外部字幕と内蔵字幕を切り替えられる。
4. **Given** 動画ファイルが未選択の状態で、**When** 字幕ファイルを開こうとすると、**Then** エラーメッセージが表示され操作が無効化される。
5. **Given** 対応していない字幕ファイル形式を選択したとき、**Then** ユーザーに対応形式（SRT/ASS）を案内するエラーメッセージが表示される。

---

### User Story 2 - ウィンドウ位置・サイズの記憶（F-403）(Priority: P1)

アプリを閉じて再度開いたとき、前回終了時と同じウィンドウ位置・サイズで起動する。マルチモニタ環境でも毎回ウィンドウを移動し直す手間がなくなる。

**Why this priority**: 日常的に使うすべてのユーザーに影響する基本的なデスクトップアプリ体験。実装難易度が低く、ユーザーの毎回の手間を解消する。

**Independent Test**: アプリをドラッグしてウィンドウを移動・リサイズしてから終了し、再起動すると同じ位置・サイズで表示される。

**Acceptance Scenarios**:

1. **Given** ウィンドウを特定の位置・サイズに変更して終了したとき、**When** 次回起動すると、**Then** 前回と同じ位置・サイズで表示される。
2. **Given** マルチモニタ環境でサブモニタにウィンドウを移動して終了したとき、**When** 次回起動すると、**Then** サブモニタの同じ位置に表示される。
3. **Given** ウィンドウ位置が記憶されている状態で、**When** 「表示 > ウィンドウ位置をリセット」を選択すると、**Then** デフォルトの位置・サイズに戻る。
4. **Given** 記憶されたウィンドウ位置が現在接続されていないモニタにある場合、**When** 起動すると、**Then** プライマリモニタ上の有効な位置に調整されて表示される。

---

### User Story 3 - ツールチップの充実（F-502）(Priority: P1)

初見ユーザーまたは久しぶりに使うユーザーがボタン・コントロールにマウスを乗せると、機能の説明とキーボードショートカットが記載されたツールチップが表示される。

**Why this priority**: 機能が増えるほど発見可能性が低下しており、ツールチップ充実は最も工数対効果が高い改善。すべてのユーザーが恩恵を受ける。

**Independent Test**: `+1F`/`-1F` ボタンにマウスを乗せると日本語と英語両方に対応したツールチップが表示される。

**Acceptance Scenarios**:

1. **Given** 任意のボタン・コントロールにマウスを乗せたとき、**When** 少し待つと、**Then** そのコントロールの機能を説明するツールチップが表示される。
2. **Given** キーボードショートカットが割り当てられているコントロールのツールチップが表示されたとき、**Then** ショートカットキーが括弧付きで明記されている（例: 「A 点を 1 フレーム前にずらします (Shift+,)」）。
3. **Given** UI 言語が日本語に設定されているとき、**Then** ツールチップが日本語で表示される。
4. **Given** UI 言語が英語に設定されているとき、**Then** ツールチップが英語で表示される。
5. **Given** すべてのボタン・コントロールを確認したとき、**Then** ツールチップが設定されていないコントロールが存在しない。

---

### Edge Cases

- 字幕ファイルのパスに日本語・特殊文字が含まれる場合、正しく読み込めるか？
- ウィンドウサイズが最小サイズ以下に記憶された場合、起動時に適切なサイズに補正されるか？
- フルスクリーン状態で終了した場合、次回起動時はフルスクリーン移行前のウィンドウ位置・サイズで起動する（FR-206 で解決済み）。
- 字幕ファイルのエンコーディングが UTF-8 以外（Shift-JIS など）の場合、UTF-8 への変換を案内するエラーを表示する（FR-107 で解決済み）。

## Requirements *(mandatory)*

### Functional Requirements

**F-201 外部字幕ファイルの読み込み**

- **FR-101**: システムは「字幕ファイルを開く」メニュー項目を字幕メニュー内に追加しなければならない。
- **FR-102**: ユーザーは SRT および ASS 形式の字幕ファイルを選択して動画に付加できなければならない。
- **FR-103**: 動画未選択の状態で字幕ファイルを開こうとした場合、動画を先に開くよう案内するエラーメッセージを表示しなければならない。
- **FR-104**: 非対応ファイル形式を選択した場合、対応形式（SRT/ASS）を案内するエラーメッセージを表示しなければならない。
- **FR-107**: 字幕ファイルの読み込みに失敗した場合（エンコーディング非互換・ファイル破損等）、「UTF-8 形式の字幕ファイルを使用してください」と案内するエラーメッセージを表示しなければならない。なお、エンコーディングの詳細な検出は VLC に委任し、アプリ側は読み込み失敗を一括してエラーとして扱う。
- **FR-105**: 外部字幕は既存の内蔵字幕トラックと共存でき、字幕メニューで切り替えられなければならない。
- **FR-106**: 別の動画ファイルを開いたとき、読み込み済みの外部字幕はリセットされ、新しい動画は字幕なし状態で開始しなければならない。

**F-403 ウィンドウ位置・サイズの記憶**

- **FR-201**: システムはアプリ終了時のウィンドウ位置（x, y）とサイズ（幅, 高さ）を設定ファイルに保存しなければならない。
- **FR-202**: システムは次回起動時に保存されたウィンドウ位置・サイズを復元しなければならない。
- **FR-203**: システムは「表示 > ウィンドウ位置をリセット」メニュー項目を追加し、デフォルト値に戻せなければならない。
- **FR-204**: 記憶されたウィンドウ位置が現在の画面範囲外にある場合、プライマリモニタの有効な位置に自動調整しなければならない。
- **FR-205**: フルスクリーン移行時にその直前のウィンドウ位置・サイズを内部で保持しなければならない。フルスクリーン中に変更されたジオメトリは記憶対象外とする。
- **FR-206**: フルスクリーン状態でアプリを終了した場合、次回起動時はフルスクリーン移行前に保持していたウィンドウ位置・サイズで起動しなければならない。

**F-502 ツールチップの充実**

- **FR-301**: すべてのボタン・コントロールに機能説明のツールチップを追加しなければならない。
- **FR-302**: ショートカットキーが割り当てられているコントロールはツールチップにキーを記載しなければならない。
- **FR-303**: ツールチップ文字列は既存の i18n 機構（`t()` 関数）で日本語/英語に対応しなければならない。
- **FR-304**: ツールチップ対象には少なくとも以下を含まなければならない: 再生/停止、シークバー、音量、コマ送り (+1F/-1F)、A/B 点ボタン、ループ切り替え、ブックマーク関連ボタン（メモ・タグ・カウンター）。

### Key Entities

- **字幕ファイル**: ファイルパス、フォーマット（SRT/ASS）を持つ。動画メディアに付加される一時的な設定。動画切替時に自動クリアされる（ライフサイクルは単一動画セッションに限定）。
- **ウィンドウ状態**: x 座標、y 座標、幅、高さを持つ。`settings.json` の `window_geometry` フィールドに保存される。
- **ツールチップ文字列**: i18n キーと日本語・英語テキストのペア。`i18n.py` で管理される。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 語学学習者が手持ちの SRT ファイルを 30 秒以内にアプリに読み込み、字幕付き AB ループ練習を開始できる。
- **SC-002**: 日常ユーザーがアプリを再起動したとき、ウィンドウ位置の再調整なしに即座に作業を再開できる（ウィンドウ移動操作ゼロ）。
- **SC-003**: 初見ユーザーがマウスオーバーだけですべての主要コントロールの機能を把握できる（ショートカット一覧ダイアログを開かずに）。
- **SC-004**: UI 文字列の変更が日本語・英語どちらの言語設定でも正しく反映される（翻訳漏れゼロ）。

---

## Assumptions

- 字幕ファイルのエンコーディングは UTF-8 のみ対応とする。他エンコーディングにはエラーメッセージを表示し、UTF-8 への変換を案内する（将来の多エンコーディング対応は別途検討）。
- ウィンドウ位置はフルスクリーン状態では記憶対象外とし、ウィンドウモードの最終状態のみ保存する。
- ツールチップはすでに実装されているウィジェットへの追加のみ行い、UI 構造の変更は最小限とする。
- `settings.json` の既存スキーマを拡張する形でウィンドウ位置を保存する（新規ファイル不要）。

---

## Dependencies

- F-201 は既存のマルチトラック（字幕メニュー）機能（spec 007）を拡張する。
- F-403 は既存の `AppSettings` / `settings.json` を拡張する。
- F-502 は既存の `i18n.py` の `t()` 関数と翻訳辞書を拡張する。
- F-201 と F-502 は互いに独立しており、どちらかを先に実装できる。
